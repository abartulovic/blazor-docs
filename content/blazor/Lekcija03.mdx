## 2. Model `Animal` i zašto ga sada usklađuješ

Ako želiš prikazati životinje u azilu, moraš definirati koje informacije o svakoj životinji postoje u aplikaciji. To radiš pomoću modela.

Model je klasa koja opisuje jedan pojam iz sustava. Ovdje je to jedna životinja u azilu.

U ovoj lekciji model `Animal` treba biti bogatiji nego prije jer sada ne prikazuješ samo osnovni tekst nego i dodatne podatke, sliku i status.

Najčešća lokacija modela u ovom projektu je:


### Cjeloviti sadržaj datoteke `Animal.cs`

U ovoj lekciji nemoj kopirati samo dio modela. Ovdje se radi o punoj verziji datoteke `Animal.cs` koja zamjenjuje jednostavniju raniju verziju i usklađuje model s onim što sada prikazuješ na stranici `Animals`.

```csharp
namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Species { get; set; } = string.Empty;
    public string Breed { get; set; } = string.Empty;
    public string Gender { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public bool IsAdopted { get; set; }
    public string ImageUrl { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
}
```

### Što je dodano i zašto

- `Gender` – za prikaz spola životinje u tablici
- `ArrivalDate` – za prikaz datuma dolaska u azil
- `IsAdopted` – za prikaz statusa udomljenja
- `ImageUrl` – za prikaz slike životinje
- `Description` – za kratki opis ili napomenu

### Mjesto za slike životinja

Za probne slike pripremi folder:

```text
wwwroot/images/animals/
```

U ovoj lekciji očekuju se ove datoteke:

- `luna.webp`
- `maza.webp`
- `rex.webp`

Ako slike još nemaš, privremeno stavi placeholder slike s tim nazivima i kasnije ih zamijeni pravim fotografijama.

## 3. Izrada stranice `Animals.razor`

Sada nadograđuješ stranicu:

```text
Components/Pages/Animals.razor
```

Ovdje je važno razlikovati dvije stvari:

- u prethodnoj lekciji imao si jednostavniji statički prikaz
- u ovoj lekciji taj prikaz nadograđuješ u pravu stranicu s `MudTable` tablicom, slikama i statusima

To znači da ne krećeš od nule nego prepravljaš i proširuješ postojeću ideju.

U nastavku prvo ide objašnjenje po manjim dijelovima. Tek nakon toga dolazi cjeloviti kod stranice.

### 3.1 Ruta i osnovni zaglavni dio stranice

Na vrh stranice dodaj rutu, `using` direktivu i osnovni naslov s uvodnim tekstom.

```razor
@page "/animals"
@using AzilEdu.Shared.Models

<PageTitle>Životinje</PageTitle>

<MudText Typo="Typo.h4" Class="mb-2">Popis životinja</MudText>
<MudText Typo="Typo.body1" Class="mb-4">
    Ovdje se prikazuje pregled životinja koje se trenutačno nalaze u azilu.
</MudText>
```

Ovaj dio:

- određuje URL stranice
- omogućuje korištenje modela `Animal`
- prikazuje naslov i kratki uvod

### 3.2 Što se nadograđuje iz prethodne lekcije

U ranijem prikazu podaci su mogli biti ispisani jednostavno, bez tablice.

```razor
<MudPaper Class="pa-4">
    <MudText Typo="Typo.subtitle1">Luna — Pas — Labrador</MudText>
    <MudText Typo="Typo.subtitle1">Maza — Mačka — Domaća kratkodlaka</MudText>
    <MudText Typo="Typo.subtitle1">Rex — Pas — Njemački ovčar</MudText>
</MudPaper>
```

Sada taj dio zamjenjuješ bogatijim tabličnim prikazom.

### 3.3 Novi `MudTable` prikaz

Umjesto jednostavnog ispisa sada dodaj `MudTable`.

```razor
<MudTable Items="@animals" Hover="true" Striped="true" Bordered="true" Dense="true">
    <HeaderContent>
        <MudTh>Slika</MudTh>
        <MudTh>Ime</MudTh>
        <MudTh>Vrsta</MudTh>
        <MudTh>Pasmina</MudTh>
        <MudTh>Spol</MudTh>
        <MudTh>Starost</MudTh>
        <MudTh>Datum dolaska</MudTh>
        <MudTh>Status</MudTh>
    </HeaderContent>
    <RowTemplate>
        <MudTd DataLabel="Slika">
            <MudAvatar Size="Size.Large" Image="@context.ImageUrl" Alt="@context.Name" />
        </MudTd>
        <MudTd DataLabel="Ime">@context.Name</MudTd>
        <MudTd DataLabel="Vrsta">@context.Species</MudTd>
        <MudTd DataLabel="Pasmina">@context.Breed</MudTd>
        <MudTd DataLabel="Spol">@context.Gender</MudTd>
        <MudTd DataLabel="Starost">@context.Age</MudTd>
        <MudTd DataLabel="Datum dolaska">@context.ArrivalDate.ToShortDateString()</MudTd>
        <MudTd DataLabel="Status">
            @if (context.IsAdopted)
            {
                <MudChip T="string" Color="Color.Success" Variant="Variant.Filled">
                    Udomljena
                </MudChip>
            }
            else
            {
                <MudChip T="string" Color="Color.Warning" Variant="Variant.Filled">
                    U azilu
                </MudChip>
            }
        </MudTd>
    </RowTemplate>
</MudTable>
```

Ovdje dodaješ nekoliko važnih novosti:

- `MudAvatar` prikazuje sliku životinje
- novi stupci daju više informacija nego prije
- `MudChip` vizualno ističe status
- `T="string"` je potreban jer je `MudChip` generička komponenta pa Razor ovdje ne može sam zaključiti tip samo iz teksta unutar chipa

### Zašto kod `MudChip` koristiš `T="string"`

`MudChip` je generička komponenta. To znači da u pozadini ne postoji samo jedan `MudChip`, nego tipizirana verzija komponente koja očekuje da generic tip bude poznat već tijekom kompilacije.

U ovom primjeru unutar `MudChip` piše samo tekst:

```razor
<MudChip T="string" Color="Color.Success" Variant="Variant.Filled">
    Udomljena
</MudChip>
```

Kad bi `T="string"` izostavio, Razor bi vidio vizualne postavke poput `Color` i `Variant` te običan tekst kao sadržaj, ali iz toga ne može sigurno zaključiti koji je generički tip komponente. Zato javlja grešku da se tip ne može inferirati.

Dakle, `T="string"` ne stavljaš samo da "utišaš" grešku, nego da jasno kažeš:

- ova instanca `MudChip` radi sa string tipom
- generički tip komponente mora biti poznat pri kompilaciji
- Razor ga ovdje ne može sam zaključiti jer ne postoji posebno tipizirano svojstvo iz kojeg bi to pročitao

To je korisno zapamtiti i za druge generičke MudBlazor komponente: kad Razor ne može sam zaključiti tip, moraš ga zadati ručno.

## 7. Domaći rad: izradi modul `HousingUnits`

Za domaći ne radiš ponovno `Animals`. Trebaš primijeniti isti obrazac rada na drugi modul projekta: `HousingUnits`.

To znači da moraš samostalno napraviti novu tablicu i novu stranicu, ali po istoj logici koju si vidio na satu.

### Datoteke koje trebaš napraviti ili urediti

1. `Shared/Models/HousingUnit.cs`
2. `Components/Pages/HousingUnits.razor`
3. navigaciju u kojoj dodaješ link na `/housing-units`

### Model `HousingUnit` treba sadržavati ova svojstva

```csharp
public class HousingUnit
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string UnitType { get; set; } = string.Empty;
    public int Capacity { get; set; }
    public int Occupied { get; set; }
    public bool IsFull { get; set; }
    public string ImageUrl { get; set; } = string.Empty;
}
```

### Što moraš napraviti

1. napravi model `HousingUnit` u `Shared/Models/HousingUnit.cs`
2. napravi stranicu `HousingUnits.razor` s rutom `@page "/housing-units"`
3. dodaj naslov stranice `Smještajne jedinice`
4. dodaj kratak uvodni tekst ispod naslova
5. dodaj lokalnu listu s točno 4 probna zapisa u `@code` dio stranice
6. pripremi 4 slike i smjesti ih u folder `wwwroot/images/housing-units/`
7. prikaži podatke pomoću `MudTable`
8. u tablici obavezno prikaži stupce:
   - slika
   - naziv jedinice
   - tip jedinice
   - kapacitet
   - trenutno zauzeto
   - status
9. status prikaži pomoću `MudChip`
   - `Puno` ako je `IsFull == true`
   - `Slobodno` ako je `IsFull == false`
10. dodaj link u navigaciju koji vodi na `/housing-units`
11. provjeri otvara li se nova stranica bez greške

### Nazivi slika koje se očekuju

- `box-1.webp`
- `box-2.webp`
- `box-3.webp`
- `box-4.webp`

### Kako gotov rad treba izgledati

Gotov domaći rad mora imati:

- ispravnu rutu
- naslov i uvodni tekst
- tablicu s točno 4 retka
- stupac sa slikama
- jasno prikazan kapacitet i trenutno zauzeće
- status prikazan pomoću `MudChip`
- dodanu poveznicu u navigaciji
- izgled usklađen s ostatkom aplikacije
