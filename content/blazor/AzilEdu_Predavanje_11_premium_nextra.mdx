## Cilj predavanja

Cilj ove lekcije je proširiti AzilEdu tako da za svaku životinju možemo:

- spremiti više medicinskih zapisa
- zabilježiti datum pregleda ili tretmana
- dodati opis, terapiju i napomenu
- prikazati povijest zapisa na details stranici životinje
- dodati novi zapis kroz zasebnu formu ili ugrađenu sekciju
- obrisati zapis ako je unesen pogrešno

Na kraju ove lekcije aplikacija više nije samo katalog životinja, nego počinje nalikovati stvarnom informacijskom sustavu za azil.

## Zašto je ovaj modul važan?

U praksi azil mora pratiti mnogo više od osnovnih podataka o životinji. Djelatnici trebaju znati:

- je li životinja bila na pregledu
- je li primila terapiju
- ima li otvorene zdravstvene probleme
- kada je zadnji put pregledana
- postoje li posebne napomene koje treba uzeti u obzir prije udomljenja

Bez toga aplikacija ostaje korisna samo za osnovni pregled podataka. S medicinskom evidencijom projekt postaje mnogo ozbiljniji.

<InfoBox type="success" title="Didaktička ideja ove lekcije">
  Učenici prvi put rade modul u kojem jedna životinja ima <strong>više povezanih zapisa</strong>. To je važan iskorak iz jednostavnog CRUD-a nad jednom tablicom.
</InfoBox>

## Novi entitet: `MedicalRecord`

Za ovaj modul uvodimo novu tablicu. Jedna životinja može imati više medicinskih zapisa, što znači da radimo relaciju:

- `Animal` → jedan
- `MedicalRecord` → više

### Predložena svojstva klase

```csharp
public class MedicalRecord
{
    public int Id { get; set; }

    public int AnimalId { get; set; }
    public Animal? Animal { get; set; }

    public DateTime RecordDate { get; set; } = DateTime.Today;

    public string Title { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
    public string? Therapy { get; set; }
    public string? VeterinarianName { get; set; }
    public bool IsResolved { get; set; }
}
```

### Objašnjenje polja

- `AnimalId` povezuje zapis s točno jednom životinjom
- `RecordDate` govori kada je pregled ili događaj zabilježen
- `Title` služi za kratak naziv zapisa, npr. “Cijepljenje”, “Kontrolni pregled”, “Liječenje uha”
- `Description` opisuje što se dogodilo
- `Therapy` čuva terapiju ili preporuku
- `VeterinarianName` je korisno polje za kasniji pregled povijesti
- `IsResolved` označava je li problem riješen

<InfoBox type="warning" title="Nemojte pretjerati s modelom u ovoj fazi">
  U ovoj lekciji cilj nije napraviti savršeni medicinski sustav, nego razumjeti kako se radi modul s više povezanih zapisa. Zato model držimo dovoljno bogatim da bude koristan, ali ne toliko velikim da učenici izgube nit.
</InfoBox>

## Proširenje servisa za rad s medicinskim zapisima

Do sada već imaš `AnimalService`. Sada možeš napraviti novi servis, npr. `MedicalRecordService`, ili logiku dodati u postojeći servis ako još ne želite previše klasa.

Moj savjet: uvedi poseban servis. To je urednije.

### Sučelje servisa

```csharp
public interface IMedicalRecordService
{
    Task<List<MedicalRecord>> GetByAnimalIdAsync(int animalId);
    Task AddAsync(MedicalRecord record);
    Task DeleteAsync(int id);
}
```

### Implementacija servisa

```csharp
public class MedicalRecordService : IMedicalRecordService
{
    private readonly AzilEduDbContext _db;

    public MedicalRecordService(AzilEduDbContext db)
    {
        _db = db;
    }

    public async Task<List<MedicalRecord>> GetByAnimalIdAsync(int animalId)
    {
        return await _db.MedicalRecords
            .Where(r => r.AnimalId == animalId)
            .OrderByDescending(r => r.RecordDate)
            .ToListAsync();
    }

    public async Task AddAsync(MedicalRecord record)
    {
        _db.MedicalRecords.Add(record);
        await _db.SaveChangesAsync();
    }

    public async Task DeleteAsync(int id)
    {
        var record = await _db.MedicalRecords.FindAsync(id);
        if (record is null)
            return;

        _db.MedicalRecords.Remove(record);
        await _db.SaveChangesAsync();
    }
}
```

### Registracija servisa

U `Program.cs` dodaj:

```csharp
builder.Services.AddScoped<IMedicalRecordService, MedicalRecordService>();
```

## Proširenje details stranice životinje

Ovo je srce lekcije. Na details stranici sada više ne prikazujemo samo osnovne podatke, nego i povijest medicinskih zapisa.

### Dohvat životinje s povezanim zapisima

Ako na details stranici dohvaćaš životinju direktno iz baze, proširi upit:

```csharp
animal = await _db.Animals
    .Include(a => a.Species)
    .Include(a => a.Breed)
    .Include(a => a.AnimalStatus)
    .Include(a => a.MedicalRecords)
    .FirstOrDefaultAsync(a => a.Id == id);
```

Ako koristiš servis, istu logiku stavi u servisnu metodu.

<InfoBox type="info" title="Zašto koristimo `Include(a => a.MedicalRecords)`?">
  Zato što želimo da EF Core učita i povezane medicinske zapise uz životinju. Bez toga details stranica ne bi imala podatke o pregledima i tretmanima.
</InfoBox>

## Dodavanje novog medicinskog zapisa

Sada trebamo omogućiti i unos novog zapisa. To možemo napraviti na dva načina:

1. zasebna stranica, npr. `/animals/{id}/medical/new`
2. sekcija unutar details stranice

Za učenike je uredniji prvi pristup, jer jasnije odvaja prikaz i unos.

### Nova Razor stranica

Predložena putanja:

```text
Components/Pages/Animals/Medical/New.razor
```

Ili, ovisno o tvojoj strukturi projekta, druga odgovarajuća lokacija.

### Parametar životinje

```razor
@page "/animals/{AnimalId:int}/medical/new"
@inject IMedicalRecordService MedicalRecordService
@inject NavigationManager NavigationManager
@inject ISnackbar Snackbar

<PageTitle>Novi medicinski zapis</PageTitle>
```

### Kod-behind logika i model forme

```razor
@code {
    [Parameter]
    public int AnimalId { get; set; }

    private MedicalRecord model = new();

    protected override void OnInitialized()
    {
        model.AnimalId = AnimalId;
        model.RecordDate = DateTime.Today;
    }

    private async Task SaveAsync()
    {
        await MedicalRecordService.AddAsync(model);
        Snackbar.Add("Medicinski zapis je uspješno dodan.", Severity.Success);
        NavigationManager.NavigateTo($"/animals/{AnimalId}/details");
    }
}
```

### Primjer metode za brisanje

```csharp
private async Task DeleteRecordAsync(int recordId)
```


## Validacija modela

Ako želiš učenicima pokazati bolji pristup, u model možeš dodati DataAnnotations atribute.

```csharp
using System.ComponentModel.DataAnnotations;

public class MedicalRecord
{
    public int Id { get; set; }

    public int AnimalId { get; set; }
    public Animal? Animal { get; set; }

    [Required]
    public DateTime RecordDate { get; set; } = DateTime.Today;

    [Required]
    [StringLength(100)]
    public string Title { get; set; } = string.Empty;

    [Required]
    [StringLength(1000)]
    public string Description { get; set; } = string.Empty;

    [StringLength(300)]
    public string? Therapy { get; set; }

    [StringLength(100)]
    public string? VeterinarianName { get; set; }

    public bool IsResolved { get; set; }
}
```

### Zašto je to korisno?

Jer učenik vidi da forma ne služi samo za unos podataka, nego i za očuvanje kvalitete podataka u sustavu.
