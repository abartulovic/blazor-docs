## Ishodi lekcije

Nakon ove lekcije učenik treba moći:

- objasniti razliku između **spremanja slike u bazu** i **spremanja slike na disk uz putanju u bazi**
- dodati novo svojstvo za putanju slike u model `Animal`
- napraviti migraciju za novu kolonu u tablici `Animals`
- omogućiti upload slike kroz Blazor formu
- spremiti sliku u `wwwroot/images/animals`
- prikazati stvarnu fotografiju ili fallback sliku ako fotografija ne postoji
- proširiti listu i details prikaz tako da koriste fotografiju životinje

## Zašto ne spremamo cijelu sliku direktno u bazu?

Početnici često pomisle da je najjednostavnije sliku spremiti kao binarne podatke izravno u tablicu. To tehnički jest moguće, ali za naš projekt to **nije dobar početni pristup**.

Bolje je:

- datoteku slike spremiti na server, unutar projekta
- u bazu spremiti samo **relativnu putanju do slike**

Prednosti takvog pristupa:

- baza ostaje jednostavnija
- lakše je prikazati slike u UI-u
- lakše je provjeriti postoji li slika
- projekt je didaktički jasniji za učenike

<InfoBox type="success" title="Pravilo koje ćemo pratiti">
  U ovoj lekciji **slika se sprema na disk**, a u bazi pamtimo samo tekstualnu putanju, npr. `/images/animals/5-rex.webp`.
</InfoBox>

## Korak 1 — Proširenje modela `Animal`

Prvo trebamo modelu dodati svojstvo koje će pamtiti putanju fotografije.

Ako u prethodnim lekcijama klasu `Animal` držiš u `Shared` projektu, nastavi istim obrascem. To je i dalje dobro mjesto za osnovni model koji koriste i App i Api sloj.

Dodaj novo nullable svojstvo:

```csharp
public string? PhotoPath { get; set; }
```

Cijela klasa može izgledati otprilike ovako:

```csharp
public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public string? Description { get; set; }

    public int SpeciesId { get; set; }
    public Species? Species { get; set; }

    public int? BreedId { get; set; }
    public Breed? Breed { get; set; }

    public AnimalStatus? AnimalStatus { get; set; }

    public string? PhotoPath { get; set; }

```

<InfoBox type="info" title="Zašto je `PhotoPath` nullable?">
  Zato što neke životinje u početku možda još nemaju fotografiju. Ako bismo tražili obaveznu vrijednost za svaku životinju, stvarali bismo nepotrebne probleme pri unosu i seedanju podataka.

## Korak 3 — Kako ćemo raditi upload u Blazoru?

U Blazoru se za upload datoteke tipično koristi komponenta `InputFile`. Ona omogućuje da korisnik u pregledniku odabere datoteku, a mi zatim u kodu pročitamo sadržaj i spremimo ga tamo gdje želimo.

U ovoj lekciji radit ćemo ovako:

1. korisnik na formi odabere sliku
2. datoteka se privremeno pročita
3. aplikacija generira sigurno ime datoteke
4. datoteka se spremi u `wwwroot/images/animals`
5. putanja slike se zapiše u `PhotoPath`
6. objekt `Animal` spremi se ili ažurira u bazi

## Korak 4 — Dodavanje servisa za rad s datotekama

Ne bih preporučio da sav upload kod zatrpamo izravno u `.razor` stranicu. Bolje je izdvojiti ga u zaseban servis, jer će tako kod biti čišći i lakši za održavanje.

Napravimo servis, npr. `ImageStorageService`.

Predložena lokacija:

```text
AzilEdu.App/Services/ImageStorageService.cs
```

Primjer implementacije:

```csharp
using Microsoft.AspNetCore.Components.Forms;

namespace AzilEdu.App.Services;

public class ImageStorageService
{
    private readonly IWebHostEnvironment _environment;

    public ImageStorageService(IWebHostEnvironment environment)
    {
        _environment = environment;
    }

    public async Task<string?> SaveAnimalPhotoAsync(IBrowserFile? file, int animalId)
    {
        if (file is null)
            return null;

        var uploadsFolder = Path.Combine(_environment.WebRootPath, "images", "animals");

        if (!Directory.Exists(uploadsFolder))
            Directory.CreateDirectory(uploadsFolder);

        var extension = Path.GetExtension(file.Name);
        var safeExtension = string.IsNullOrWhiteSpace(extension) ? ".webp" : extension.ToLowerInvariant();
        var fileName = $"{animalId}-{Guid.NewGuid()}{safeExtension}";
        var fullPath = Path.Combine(uploadsFolder, fileName);

        await using var stream = file.OpenReadStream(maxAllowedSize: 5 * 1024 * 1024);
        await using var fileStream = new FileStream(fullPath, FileMode.Create);
        await stream.CopyToAsync(fileStream);

        return $"/images/animals/{fileName}";
    }
}
```

<InfoBox type="warning" title="Zašto ograničavamo veličinu datoteke?">
  Ne želimo dopustiti da korisnik bez kontrole pošalje ogromnu datoteku. Zato koristimo `maxAllowedSize`. U ovoj lekciji koristimo jednostavno ograničenje od 5 MB, što je sasvim dovoljno za školski projekt i dobar uvod u sigurnije rukovanje uploadom.
</InfoBox>

## Korak 7 — Privatna varijabla za odabranu datoteku

U code dijelu stranice dodaj privatnu varijablu:

```csharp
private IBrowserFile? _selectedPhoto;
```

Zatim metodu koja sprema odabranu datoteku u tu varijablu:

```csharp
private void HandlePhotoSelected(InputFileChangeEventArgs e)
{
    _selectedPhoto = e.File;
}
```

Ako želiš, možeš odmah dodati i informativnu poruku:

```csharp
private void HandlePhotoSelected(InputFileChangeEventArgs e)
{
    _selectedPhoto = e.File;

    if (_selectedPhoto is not null)
    {
        Snackbar.Add($"Odabrana je datoteka: {_selectedPhoto.Name}", Severity.Info);
    }
}
```

## Korak 8 — Kako spremiti fotografiju kod novog unosa?

Ovdje trebamo biti pažljivi. Kod **novog unosa** `animal.Id` još ne postoji dok zapis ne spremimo u bazu. Zato imamo dva razumna pristupa:

### Pristup A — prvo spremi zapis, pa zatim spremi sliku
Ovo je najjednostavnije za učenike:

1. spremi `Animal` bez slike
2. baza dodijeli `Id`
3. spremi sliku koristeći dobiveni `Id`
4. ažuriraj `PhotoPath`
5. ponovno spremi promjenu

To je pristup koji ćemo koristiti u ovoj lekciji.

Primjer metode za spremanje novog zapisa:

```csharp
private async Task SaveAsync()
{
    try
    {
        await AnimalService.CreateAsync(Model);

        if (_selectedPhoto is not null)
        {
            var photoPath = await ImageStorageService.SaveAnimalPhotoAsync(_selectedPhoto, Model.Id);
            Model.PhotoPath = photoPath;
            await AnimalService.UpdateAsync(Model);
        }

        Snackbar.Add("Životinja je uspješno spremljena.", Severity.Success);
        Navigation.NavigateTo("/animals");
    }
    catch (Exception ex)
    {
        Snackbar.Add($"Greška pri spremanju: {ex.Message}", Severity.Error);
    }
}
```

<InfoBox type="info" title="Zašto spremamo dvaput?">
  Zato što kod novog unosa najprije trebamo dobiti ID životinje. Tek tada možemo generirati smislen naziv datoteke i zapisati `PhotoPath`. U ozbiljnijoj aplikaciji ovo se može riješiti sofisticiranije, ali za učenje je ovaj pristup pregledan i didaktički jasan.
</InfoBox>

## Korak 13 — Treba li brisati staru sliku kad korisnik doda novu?

U ovoj lekciji možemo ostaviti jednostavniji pristup: kad korisnik doda novu sliku, spremi se nova datoteka i `PhotoPath` pokazuje na nju. Stara datoteka može ostati na disku.

To nije idealno za produkciju, ali je sasvim prihvatljivo za trenutni stupanj učenja. Kasnije možeš napraviti dodatnu lekciju o:

- brisanju stare slike
- provjeri postoji li datoteka
- čišćenju siročadi na disku

<InfoBox type="warning" title="Što još ne rješavamo?">
  U ovoj lekciji još ne radimo napredne provjere tipa MIME type validacija, resize slike, crop, optimizaciju formata ili automatsko brisanje stare datoteke. To su dobre teme, ali za sada bi samo zamutile osnovnu logiku učenja.
</InfoBox>

## Korak 14 — Najčešće pogreške učenika

### 1. Folder za slike ne postoji
Ako `wwwroot/images/animals` ne postoji, spremanje slike može pasti. Zato servis provjerava postoji li folder i po potrebi ga kreira.

### 2. Putanja slike je kriva
Ako vratiš fizičku Windows putanju umjesto web putanje, slika se neće prikazati u pregledniku. U UI-u treba biti web putanja oblika:

```text
/images/animals/neka-slika.webp
```

### 3. Učenik zaboravi migraciju
Ako `PhotoPath` postoji u klasi, ali nije dodana migracija, baza neće imati novu kolonu.

### 4. `InputFile` je dodan, ali se odabrana datoteka nigdje ne sprema
Sama kontrola nije dovoljna. Trebaš i:
- privatnu varijablu
- handler metodu
- poziv servisa tijekom create ili update procesa

### 5. Aplikacija pokušava učitati sliku koja ne postoji
Zato koristimo fallback sliku.
