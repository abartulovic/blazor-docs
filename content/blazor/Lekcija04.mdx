### Cjeloviti sadržaj datoteke `Animal.cs`

```csharp
namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Species { get; set; } = string.Empty;
    public string Breed { get; set; } = string.Empty;
    public string Gender { get; set; } = string.Empty;
    public int? Age { get; set; }
    public DateTime? ArrivalDate { get; set; }
    public bool IsAdopted { get; set; }
    public string ImageUrl { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
}
```

### Zašto je ovaj model važan za bazu

Svako javno svojstvo iz ove klase EF Core može pretvoriti u stupac tablice. To znači:

- `Id` postaje primarni ključ
- `Name`, `Species`, `Breed` i ostala tekstualna svojstva postaju stupci za tekst
- `Age` postaje numerički stupac i može ostati prazan ako dob nije poznata
- `ArrivalDate` postaje datum i može ostati prazan ako točan datum dolaska još nije zabilježen
- `IsAdopted` postaje logička vrijednost

<InfoBox type="info" title="Najvažnija mentalna poveznica">
Klasa `Animal` još nije baza. To je model u aplikaciji. Tek kada taj model povežeš s `DbContext` klasom i pokreneš migraciju, EF Core može iz njega izvesti strukturu tablice u bazi.
</InfoBox>

<InfoBox type="info" title="Poveznica sa sljedećom lekcijom">
U ovoj lekciji radimo s <strong>entity modelom `Animal`</strong> i povezujemo ga s bazom preko `DbContext` klase. U sljedećoj lekciji nećemo frontend spajati direktno na bazu, nego uvodimo <strong>DTO modele, API kontroler i HTTP dohvat podataka</strong>. Zato je važno da `Animal` već sada bude u konačnoj, usklađenoj verziji koju ćemo dalje koristiti.

### 6.1 `using` direktive i namespace

```csharp
using AzilEdu.Shared.Models;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Data;
```

Ovaj dio omogućuje:

- korištenje modela `Animal` iz Shared projekta
- korištenje EF Core klase `DbContext`
- pravilno smještanje nove klase u namespace API projekta

### 6.2 Nasljeđivanje `DbContext` klase

```csharp
public class AzilEduDbContext : DbContext
```

Ovdje govoriš da je `AzilEduDbContext` posebna klasa za rad s bazom i da nasljeđuje sve osnovne mogućnosti koje daje EF Core.

### 6.3 Konstruktor

```csharp
public AzilEduDbContext(DbContextOptions<AzilEduDbContext> options)
    : base(options)
{
}
```

Ovaj konstruktor prima konfiguraciju baze izvana, najčešće iz `Program.cs` datoteke. To je standardan način rada u modernim .NET aplikacijama.

### 6.4 `DbSet<Animal>`

```csharp
public DbSet<Animal> Animals => Set<Animal>();
```

Ova linija govori EF Coreu da u bazi želimo imati skup zapisa tipa `Animal`.

Drugim riječima:

- model `Animal` opisuje jedan zapis
- `DbSet<Animal>` predstavlja kolekciju tih zapisa
- u bazi to najčešće postaje tablica `Animals`

### Cjeloviti sadržaj datoteke `AzilEduDbContext.cs`

```csharp
using AzilEdu.Shared.Models;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Data;

public class AzilEduDbContext : DbContext
{
    public AzilEduDbContext(DbContextOptions<AzilEduDbContext> options)
        : base(options)
    {
    }

    public DbSet<Animal> Animals => Set<Animal>();
}
```

## 8. Swagger: instalacija, uključivanje i pokretanje

Prije nego u sljedećim lekcijama krenemo na kontrolere i HTTP pozive, korisno je odmah pripremiti Swagger. Swagger služi za testiranje API endpointa kroz preglednik. To znači da kasnije možeš provjeriti radi li `GET`, `POST`, `PUT` ili `DELETE` i prije nego što ih pozoveš iz `AzilEdu.App` projekta.

<InfoBox type="info" title="Zašto Swagger uvodimo već sada?">
Iako u ovoj lekciji još ne radimo kontrolere, Swagger je dobar alat za backend dio projekta. Kasnije ćeš njime lakše provjeravati vraća li API podatke ispravno i bez pogrešaka.
</InfoBox>

### 8.1 Provjeri je li Swagger već uključen

U novijim ASP.NET Core Web API projektima Swagger je često već postavljen automatski. Zato prvo otvori datoteku:


```csharp
{
    app.UseSwagger();
    app.UseSwaggerUI();
}
```

U **Package Manager Console** provjeri da je **Default project** postavljen na `AzilEdu.Api`, pa zatim pokreni:


### 8.4a Što je API pipeline i čemu služi

Kada se API aplikacija pokrene, svaki HTTP zahtjev ne ide odmah direktno u kontroler. Prvo prolazi kroz **API pipeline**.

API pipeline možeš zamisliti kao niz koraka kroz koje zahtjev prolazi prije nego što dođe do odgovarajuće akcije u kontroleru i prije nego što odgovor ode natrag prema pregledniku ili Blazor aplikaciji.

U `Program.cs` upravo tim redoslijedom slažemo ponašanje aplikacije.

Najvažnije je razumjeti da redoslijed tih linija nije slučajan. Svaka od njih uključuje dio obrade zahtjeva.

```csharp
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();
```

Što ovdje radi svaki dio:

- `app.UseSwagger()` uključuje generiranje Swagger JSON opisa API-ja
- `app.UseSwaggerUI()` uključuje web sučelje za pregled i testiranje endpointa
- `app.UseHttpsRedirection()` preusmjerava HTTP zahtjeve na HTTPS verziju
- `app.UseAuthorization()` priprema provjeru prava pristupa ako ih kasnije uvedemo
- `app.MapControllers()` govori aplikaciji da potraži rute definirane u kontrolerima i spoji ih na odgovarajuće URL adrese

<InfoBox type="info" title="Najkraće objašnjenje pipeline-a">
Pipeline je put kojim HTTP zahtjev prolazi kroz aplikaciju. Na tom putu aplikacija odlučuje treba li uključiti Swagger, preusmjeriti zahtjev na HTTPS, provjeriti pravila pristupa i na kraju pozvati odgovarajući kontroler.
</InfoBox>

Zato u lekcijama često spominjemo da nešto "ide kroz pipeline". To ne znači novu vrstu klase, nego redoslijed obrade HTTP zahtjeva u ASP.NET Core aplikaciji.

## 12. Domaći rad: samostalno dodaj tablicu `HousingUnits`

Za domaći ne ponavljamo opet `Animals`, nego isti obrazac samostalno prenosimo na novu tablicu. Odabir je ovdje namjerno strogo zadan kako bi zadatak bio jasan, a ono što ovdje napraviš nastavljamo koristiti i u sljedećim lekcijama kada uvedemo API dohvat i CUD operacije.

### 12.1 Napravi model `HousingUnit`

U projektu `AzilEdu.Shared`, u mapi `Models`, dodaj novu datoteku:

```text
Shared/Models/HousingUnit.cs
```

Model treba sadržavati točno ova svojstva:

```csharp
namespace AzilEdu.Shared.Models;

public class HousingUnit
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string UnitType { get; set; } = string.Empty;
    public int Capacity { get; set; }
    public int CurrentOccupancy { get; set; }
    public bool IsIndoor { get; set; }
}
```

### 12.2 Nadogradi `AzilEduDbContext`

U `AzilEduDbContext.cs` dodaj još jedan `DbSet`:

```csharp
public DbSet<HousingUnit> HousingUnits => Set<HousingUnit>();
```

### 12.3 Napravi novu migraciju

Nemoj ponovno koristiti `InitialCreate`. Sada radiš novu migraciju. U Package Manager Console, uz odabran `AzilEdu.Api`, pokreni:

```powershell
Add-Migration AddHousingUnits
```

### 12.4 Primijeni promjenu na bazu

U istoj Package Manager Console pokreni:

```powershell
Update-Database
```

### 12.5 Što na kraju mora postojati

Na kraju domaćeg treba postojati:

- novu datoteku `HousingUnit.cs`
- nadograđen `AzilEduDbContext`
- novu migraciju `AddHousingUnits`
- ažuriranu SQLite bazu
- objašnjenje vlastitim riječima što se promijenilo u bazi nakon dodavanja nove tablice
