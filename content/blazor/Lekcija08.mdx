<InfoBox type="info" title="Zašto sada radimo relacije?">
Do sada je aplikacija već pokazala da može prikazivati, dodavati, uređivati i brisati životinje. Međutim, ako su <code>Species</code>, <code>Breed</code> i <code>Status</code> samo običan tekst u tablici <code>Animals</code>, projekt vrlo brzo postaje neuredan.

Primjer problema:

- jednom učenik upiše <code>Pas</code>
- drugi upiše <code>pas</code>
- treći upiše <code>PAS</code>
- netko napiše <code>Mješanac</code>, a netko <code>Mjesanac</code>

Baza tada tehnički radi, ali podaci više nisu pouzdani. Zato uvodimo zasebne tablice i relacije.
</InfoBox>

## Što ćemo točno promijeniti u modelu

U ovoj lekciji uvodimo tri nove pomoćne tablice:

1. <code>Species</code> — vrsta životinje, npr. pas, mačka, kunić
2. <code>Breed</code> — pasmina, npr. labrador, perzijska mačka, mješanac
3. <code>AnimalStatus</code> — status životinje, npr. dostupna, rezervirana, udomljena, na liječenju

Nakon toga tablica <code>Animals</code> više neće spremati samo tekst, nego i identifikatore povezanih zapisa.

<InfoBox type="success" title="Važna ideja ove lekcije">
Jedna stranica u aplikaciji ne mora prikazivati podatke samo iz jedne tablice. U ozbiljnijim aplikacijama korisnik često gleda **jedan objedinjeni prikaz**, a podaci dolaze iz više povezanih tablica.
</InfoBox>

## Logika relacija

Prije koda učenici trebaju razumjeti **zašto** radimo baš ovakve veze.

### 1. Species

Jedna vrsta može pripadati mnogim životinjama.

Primjer:

- vrsta <code>Pas</code> može imati 20 različitih životinja
- vrsta <code>Mačka</code> može imati 15 različitih životinja

To znači:

- jedna <code>Species</code>
- više <code>Animals</code>

### 2. Breed

Jedna pasmina također može pripadati mnogim životinjama.

Primjer:

- više pasa mogu biti iste pasmine
- više mačaka mogu biti iste pasmine

### 3. AnimalStatus

Jedan status može biti dodijeljen mnogim životinjama.

Primjer:

- više životinja može biti <code>Dostupna za udomljenje</code>
- više životinja može biti <code>Na liječenju</code>

### 4. Breed i Species

Ako želiš uredniji model, svaka pasmina može pripadati određenoj vrsti.

Primjer:

- pasmina <code>Labrador</code> pripada vrsti <code>Pas</code>
- pasmina <code>Britanska kratkodlaka</code> pripada vrsti <code>Mačka</code>

To znači da i između <code>Breed</code> i <code>Species</code> možemo imati relaciju.

## Proširenje `DbContexta`

U `AzilEduDbContext` trebaš dodati nove skupove podataka.

```csharp
using Microsoft.EntityFrameworkCore;

public class AzilEduDbContext : DbContext
{
    public AzilEduDbContext(DbContextOptions<AzilEduDbContext> options)
        : base(options)
    {
    }

    public DbSet<Animal> Animals => Set<Animal>();
    public DbSet<Species> Species => Set<Species>();
    public DbSet<Breed> Breeds => Set<Breed>();
    public DbSet<AnimalStatus> AnimalStatuses => Set<AnimalStatus>();
}
```

Ako želiš, možeš dodatno konfigurirati odnose u metodi <code>OnModelCreating</code>.

### Primjer konfiguracije odnosa

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    base.OnModelCreating(modelBuilder);

    modelBuilder.Entity<Breed>()
        .HasOne(b => b.Species)
        .WithMany(s => s.Breeds)
        .HasForeignKey(b => b.SpeciesId)
        .OnDelete(DeleteBehavior.Restrict);

    modelBuilder.Entity<Animal>()
        .HasOne(a => a.Species)
        .WithMany(s => s.Animals)
        .HasForeignKey(a => a.SpeciesId)
        .OnDelete(DeleteBehavior.Restrict);

    modelBuilder.Entity<Animal>()
        .HasOne(a => a.Breed)
        .WithMany(b => b.Animals)
        .HasForeignKey(a => a.BreedId)
        .OnDelete(DeleteBehavior.Restrict);

    modelBuilder.Entity<Animal>()
        .HasOne(a => a.AnimalStatus)
        .WithMany(s => s.Animals)
        .HasForeignKey(a => a.AnimalStatusId)
        .OnDelete(DeleteBehavior.Restrict);
}
```

<InfoBox type="info" title="Zašto koristimo `DeleteBehavior.Restrict`?">
Ako kasnije netko pokuša obrisati vrstu, pasminu ili status koji se već koristi kod neke životinje, baza neće dopustiti brisanje bez razmišljanja. To je sigurnije ponašanje za poslovne podatke.
</InfoBox>

## Nova migracija

Budući da smo promijenili modele i proširili bazu, sada moramo napraviti novu migraciju.

### Koraci

1. spremi sve datoteke
2. provjeri da se projekt može kompajlirati
3. napravi novu migraciju
4. primijeni migraciju na bazu

```powershell
Add-Migration AddSpeciesBreedAndStatus
Update-Database
```

<InfoBox type="warning" title="Česta pogreška učenika">
Ako projekt ne prolazi build, migracija će često pasti. Zato prije migracije provjeri ima li grešaka u modelima, namespaceovima i referencama između projekata.
</InfoBox>

### Što će migracija napraviti

Migracija će:

- stvoriti nove tablice
- dodati strane ključeve u tablicu <code>Animals</code>
- povezati tablice prema modelima

Ako već imaš postojeće podatke u <code>Animals</code>, prijelaz može tražiti dodatnu prilagodbu. U nastavnom projektu često je najjednostavnije i najčišće krenuti s urednim modelom prije ozbiljnijeg punjenja podataka.

## Seed podaci za pomoćne tablice

Ako želiš da dropdownovi odmah rade smisleno, trebaš ubaciti početne podatke za vrste, pasmine i statuse.

Jedan pristup je ručni unos kroz bazu, ali za nastavu je jasnije pokazati seed podatke u kodu.

### Primjer seed podataka u `OnModelCreating`

```csharp
modelBuilder.Entity<Species>().HasData(
    new Species { Id = 1, Name = "Pas" },
    new Species { Id = 2, Name = "Mačka" },
    new Species { Id = 3, Name = "Kunić" }
);

modelBuilder.Entity<Breed>().HasData(
    new Breed { Id = 1, Name = "Mješanac", SpeciesId = 1 },
    new Breed { Id = 2, Name = "Labrador", SpeciesId = 1 },
    new Breed { Id = 3, Name = "Perzijska", SpeciesId = 2 },
    new Breed { Id = 4, Name = "Europska kratkodlaka", SpeciesId = 2 }
);

modelBuilder.Entity<AnimalStatus>().HasData(
    new AnimalStatus { Id = 1, Name = "Dostupna za udomljenje" },
    new AnimalStatus { Id = 2, Name = "Rezervirana" },
    new AnimalStatus { Id = 3, Name = "Udomljena" },
    new AnimalStatus { Id = 4, Name = "Na liječenju" }
);
```

<InfoBox type="success" title="Zašto je seed koristan u nastavi?">
Svi učenici dobivaju iste početne podatke. To smanjuje kaos u radu i olakšava praćenje lekcije.

## Prilagodba servisa

Ako si u prethodnoj lekciji izdvojio rad s bazom u servis, sada servis mora znati dohvatiti i povezane podatke.

### Dohvat svih životinja s relacijama

```csharp
using Microsoft.EntityFrameworkCore;

public class AnimalService
{
    private readonly AzilEduDbContext _context;

    public AnimalService(AzilEduDbContext context)
    {
        _context = context;
    }

    public async Task<List<Animal>> GetAllAsync()
    {
        return await _context.Animals
            .Include(a => a.Species)
            .Include(a => a.Breed)
            .Include(a => a.AnimalStatus)
            .OrderBy(a => a.Name)
            .ToListAsync();
    }
}
```

### Dohvat jedne životinje s relacijama

```csharp
public async Task<Animal?> GetByIdAsync(int id)
{
    return await _context.Animals
        .Include(a => a.Species)
        .Include(a => a.Breed)
        .Include(a => a.AnimalStatus)
        .FirstOrDefaultAsync(a => a.Id == id);
}
```

<InfoBox type="info" title="Zašto koristimo `Include`?">
Ako samo dohvatimo <code>Animals</code>, često nećemo automatski dobiti popunjena navigacijska svojstva. Metoda <code>Include</code> govori EF Coreu da uz osnovni zapis dohvati i povezane entitete.
</InfoBox>

## Prilagodba forme za unos i uređivanje

Sada forma više ne treba imati obične tekstualne inpute za vrstu, pasminu i status. Umjesto toga koristi dropdownove.

To je idealno mjesto da učenici vide kako se baza i UI međusobno nadopunjuju.

### Što forma sada treba imati

- listu vrsta
- listu pasmina
- listu statusa
- odabir iz dropdowna
- vezanje odabrane vrijednosti na strani ključ modela

### Primjer učitavanja pomoćnih podataka

```csharp
private List<Species> speciesList = new();
private List<Breed> breedList = new();
private List<AnimalStatus> statusList = new();

protected override async Task OnInitializedAsync()
{
    speciesList = await _context.Species.OrderBy(s => s.Name).ToListAsync();
    breedList = await _context.Breeds.OrderBy(b => b.Name).ToListAsync();
    statusList = await _context.AnimalStatuses.OrderBy(s => s.Name).ToListAsync();
}
```

## Najčešće pogreške učenika

### 1. Zaboravljen `Include`

Simptom:

- tablica radi
- ali vrsta, pasmina ili status ostaju prazni

Uzrok:

- podaci iz povezanih tablica nisu dohvaćeni

### 2. Pogrešan strani ključ

Simptom:

- aplikacija sprema zapis
- ali prikazuje čudne ili neočekivane podatke

Uzrok:

- pogrešno vezanje dropdowna na model

### 3. Migracija ne prolazi

Simptom:

- `Add-Migration` javlja grešku

Uzrok:

- model nije kompajliran
- namespaceovi nisu usklađeni
- `DbContext` nema pristup svim modelima

### 4. Seed podaci i postojeći podaci se sukobljavaju

Ako projekt već ima stari sadržaj baze, ponekad je jednostavnije obrisati razvojnu bazu i ponovno izgraditi strukturu dok si još u razvojnoj fazi.

<InfoBox type="warning" title="Pedagoška napomena">
Učenicima treba jasno reći da razvojna baza nije isto što i produkcijska baza. U razvoju se baza ponekad briše i ponovno stvara dok model još nije stabilan.
</InfoBox>

   - na liječenju = crvena
3. Razmisli koje bi još pomoćne tablice trebale postojati u stvarnoj aplikaciji za azil.

## Priprema za sljedeću lekciju

Sve što smo danas napravili služi jednom vrlo konkretnom cilju:

**u sljedećoj lekciji izradit ćemo pravu stranicu detalja životinje**.

Ta stranica više neće prikazivati samo osnovna polja iz jedne tablice, nego objedinjene podatke iz više povezanih entiteta.
