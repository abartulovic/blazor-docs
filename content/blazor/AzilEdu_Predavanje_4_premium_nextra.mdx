## Zašto koristimo SQLite?

Za ovu fazu projekta koristimo **SQLite** zato što je:

- jednostavan za početno učenje
- lagan za postavljanje
- ne traži instalaciju posebnog serverskog servisa
- odličan za školski projekt i lokalni razvoj
- dovoljno dobar da učenici razumiju kako EF Core i migracije rade

Kasnije se isti princip može prenijeti i na SQL Server ili neku drugu ozbiljniju bazu.

## 1. Provjera modela `Animal`

Prije baze moramo imati model koji želimo spremati.

Ako iz prethodne lekcije već imate klasu `Animal`, sada je ponovno pregledajte i po potrebi doradite. Ako je još nemate na pravom mjestu, sada je pravo vrijeme da je uredno smjestite.

Preporuka je da je držite u projektu **`AzilEdu.Shared`**, jer će taj model trebati i frontend i backend.

### Primjer klase `Animal`

```csharp
namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Species { get; set; } = string.Empty;
    public string Breed { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public bool IsAdopted { get; set; }
}
```

<InfoBox type="info" title="Zašto je `Id` važan?">
Svojstvo `Id` je identifikator zapisa. EF Core će ga prepoznati kao **primarni ključ** tablice.  

To znači da će svaki zapis u bazi imati jedinstvenu vrijednost po kojoj ga možemo pronaći, uređivati ili obrisati.
</InfoBox>

## 4. Izrada klase `AzilEduDbContext`

Sada radimo najvažniju novu klasu ove lekcije: **DbContext**.

DbContext je klasa koja predstavlja most između naše aplikacije i baze podataka.

Ona govori EF Coreu:

- koje entitete pratimo
- koje tablice želimo imati
- kako se povezujemo s bazom

### Datoteka: `Data/AzilEduDbContext.cs`

```csharp
using AzilEdu.Shared.Models;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Data;

public class AzilEduDbContext : DbContext
{
    public AzilEduDbContext(DbContextOptions<AzilEduDbContext> options)
        : base(options)
    {
    }

    public DbSet<Animal> Animals => Set<Animal>();
}
```

## Objašnjenje ove klase red po red

### `public class AzilEduDbContext : DbContext`

Ovdje govorimo da naša klasa nasljeđuje `DbContext`.  
Time dobivamo sve osnovne mogućnosti rada s EF Coreom.

### Konstruktor s `DbContextOptions`

```csharp
public AzilEduDbContext(DbContextOptions<AzilEduDbContext> options)
    : base(options)
{
}
```

Ovo omogućuje da konfiguraciju baze pošaljemo izvana, najčešće iz `Program.cs` datoteke.

To je standardan i ispravan način rada u modernim .NET aplikacijama.

### `DbSet<Animal>`

```csharp
public DbSet<Animal> Animals => Set<Animal>();
```

Ova linija kaže:

> U bazi želimo imati kolekciju zapisa tipa `Animal`.

Drugim riječima, EF Core iz toga zaključuje da treba postojati tablica za životinje.

<InfoBox type="success" title="Važna mentalna poveznica">
- **Klasa `Animal`** predstavlja jedan objekt u aplikaciji.
- **`DbSet<Animal>`** predstavlja skup takvih objekata.
- U bazi to najčešće odgovara **tablici `Animals`**.
</InfoBox>

## 5. Dodavanje connection stringa u `appsettings.json`

Aplikacija mora znati **gdje se baza nalazi**. Zato u `appsettings.json` dodajemo connection string.

### Primjer konfiguracije

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=AzilEdu.db"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
```

### Što znači `Data Source=AzilEdu.db`?

To znači da će SQLite koristiti datoteku baze naziva **`AzilEdu.db`**.

Ako baza još ne postoji, EF Core je može stvoriti kada primijenimo migracije.

<InfoBox type="info" title="Gdje će se baza pojaviti?">
U najjednostavnijem obliku baza će se pojaviti kao datoteka u projektu ili u izlaznom runtime folderu, ovisno o tome kako pokrećeš aplikaciju i odakle izvršavaš naredbe.  

Za učenike je najvažnije da razumiju: SQLite baza je u ovom slučaju **jedna fizička `.db` datoteka**.
</InfoBox>

## 6. Registracija baze i DbContexta u `Program.cs`

Sada `AzilEdu.Api` moramo reći da koristi naš `DbContext` i SQLite bazu.

### Dodaj potrebne `using` direktive

```csharp
using AzilEdu.Api.Data;
using Microsoft.EntityFrameworkCore;
```

### Zatim registriraj DbContext

```csharp
builder.Services.AddDbContext<AzilEduDbContext>(options =>
    options.UseSqlite(builder.Configuration.GetConnectionString("DefaultConnection")));
```

### Primjer dijela `Program.cs`

```csharp
using AzilEdu.Api.Data;
using Microsoft.EntityFrameworkCore;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();

builder.Services.AddDbContext<AzilEduDbContext>(options =>
    options.UseSqlite(builder.Configuration.GetConnectionString("DefaultConnection")));

builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();
app.Run();
```

## Što znači `AddDbContext`?

Ta metoda registrira naš kontekst baze u **dependency injection** sustav aplikacije.

To znači da će kasnije:

- kontroleri
- servisi
- repozitoriji

moći dobiti `AzilEduDbContext` kao ovisnost bez ručnog stvaranja instance.

<InfoBox type="warning" title="Ne preskači objašnjenje DI sustava">
Početnici često samo kopiraju `AddDbContext(...)` bez razumijevanja.  

Objasni im jednostavno:

> Aplikacija sada zna kako napraviti objekt koji predstavlja vezu prema bazi i može ga dati dijelovima aplikacije kojima treba.
</InfoBox>

## 7. Što su migracije?

Ovo je najvažniji teorijski dio lekcije.

Migracija je **zapis promjene strukture baze** na temelju promjena koje smo napravili u modelima i DbContextu.

Drugim riječima:

- imamo klasu `Animal`
- imamo `DbSet<Animal>`
- EF Core vidi da na temelju toga treba napraviti tablicu
- migracija opisuje **kako** se baza treba promijeniti da bi odgovarala tom modelu

### Jednostavna definicija za učenike

<InfoBox type="info" title="Definicija koju učenici mogu zapamtiti">
**Migracija je datoteka koja opisuje promjenu baze podataka.**  

Kad promijenimo model, EF Core može generirati migraciju koja kaže kako baza treba izgledati nakon te promjene.
</InfoBox>

## Zašto su migracije korisne?

Zato što omogućuju da baza raste zajedno s aplikacijom.

Primjer:

- danas `Animal` ima 7 svojstava
- sutra dodamo `Weight`
- prekosutra dodamo `Description`

Svaka takva promjena može postati migracija. Na taj način baza se razvija kontrolirano, a ne ručno i kaotično.

## 8. Prva migracija

Kad su model, DbContext i connection string spremni, možemo generirati prvu migraciju.

### Varijanta 1 — Package Manager Console

Provjeri:

- **Startup project** treba biti onaj koji pokreće API
- **Default project** neka bude `AzilEdu.Api`

Zatim pokreni:

```powershell
Add-Migration InitialCreate
```

### Varijanta 2 — .NET CLI

Ako koristiš CLI, iz solution ili API konteksta naredba može izgledati ovako:

```bash
dotnet ef migrations add InitialCreate --project AzilEdu.Api
```

### Što će se dogoditi?

EF Core će napraviti mapu **`Migrations`** i u njoj generirati datoteke koje opisuju početnu strukturu baze.

Tipično ćeš dobiti:

- datoteku migracije
- designer datoteku
- snapshot modela

<InfoBox type="warning" title="Ako naredba ne radi">
Najčešći razlozi su:

- `Microsoft.EntityFrameworkCore.Tools` nije instaliran
- pokrećeš naredbu u krivom projektu
- solution se ne builda uspješno
- postoji tipfeler u namespaceu ili klasi `DbContext`
</InfoBox>

## 9. Primjena migracije na bazu

Sada imamo opis promjene baze, ali baza još možda nije stvarno stvorena. Zato koristimo naredbu:

```powershell
Update-Database
```

ili CLI varijantu:

```bash
dotnet ef database update --project AzilEdu.Api
```

### Što radi `Update-Database`?

Ona uzima migracije koje postoje u projektu i **primjenjuje ih na stvarnu bazu**.

U našem slučaju to znači da će:

- nastati SQLite baza
- u njoj biti napravljena tablica za životinje
- baza sada odgovarati našem modelu

<InfoBox type="success" title="Najvažnija razlika koju učenici moraju razumjeti">
- `Add-Migration` **stvara opis promjene**
- `Update-Database` **primjenjuje tu promjenu na stvarnu bazu**
</InfoBox>

## 11. Najčešće greške i kako ih objasniti učenicima

### Greška 1: Paket nije u pravom projektu

Ako su EF Core paketi dodani u `App` umjesto u `Api`, migracije neće raditi kako treba.

### Greška 2: Projekt ne builda

EF Core ne može generirati migraciju ako solution već ima compile error.

### Greška 3: Krivi default project u konzoli

Ako u Package Manager Console naredbu pokreneš nad pogrešnim projektom, EF Core možda neće naći `DbContext`.

### Greška 4: Nema connection stringa

Ako `DefaultConnection` nije definiran, `UseSqlite(...)` neće imati valjanu putanju prema bazi.

### Greška 5: Učenici ne razumiju razliku između modela i baze

To treba posebno naglasiti.

<InfoBox type="warning" title="Vrlo važna pedagoška napomena">
Učenici često misle da se baza „sama stvorila iz ničega“.  

Objasni im sljedeći lanac:

1. napisali smo klasu `Animal`
2. rekli smo DbContextu da postoji `DbSet<Animal>`
3. EF Core je na temelju toga izradio migraciju
4. migraciju smo primijenili na bazu
5. rezultat je stvarna tablica u bazi

Kad shvate taj niz, migracije im prestaju biti magija.
</InfoBox>

## 12. Što još ne radimo u ovoj lekciji?

Važno je i jasno reći što **još nije tema današnje lekcije**.

Danas **ne radimo**:

- unos nove životinje kroz formu
- dohvat podataka iz baze u prikaznoj stranici
- uređivanje i brisanje zapisa
- napredne relacije između više tablica
- DTO mapiranja i složenije API servise

Zašto? Zato što želimo prvo stabilno postaviti bazu i razumjeti migracije.

## 13. Što radimo u sljedećoj lekciji?

U sljedećoj lekciji napravit ćemo ono za što smo danas pripremili temelje:

- dohvat podataka iz baze
- prikaz stvarnih zapisa umjesto privremenih podataka
- ili unos nove životinje kroz formu, ovisno o odabranom redoslijedu rada

Najvažnije je da učenici sada razumiju:

> današnja lekcija nije „samo o naredbama“, nego o tome da projekt prvi put dobiva **pravu pohranu podataka**.
