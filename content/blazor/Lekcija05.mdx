## 2. Što je HTTP, što je kontroler, a što DTO

### HTTP

HTTP je način na koji jedna aplikacija traži podatke od druge aplikacije preko web zahtjeva.

U našem projektu to znači:

- App pošalje zahtjev prema API-ju
- API primi zahtjev
- API obradi zahtjev i vrati odgovor
- App primi odgovor i prikaže podatke

Najjednostavniji primjer u ovoj lekciji je:

~~~text
GET https://localhost:7220/api/animals
~~~

To znači: pošalji GET zahtjev na rutu <code>/api/animals</code> i vrati listu životinja.

### Kontroler

Kontroler je posebna klasa u API projektu koja prima HTTP zahtjeve i odlučuje što treba napraviti.

U našem slučaju kontroler:

- prima zahtjev za dohvat životinja
- koristi <code>AzilEduDbContext</code>
- čita podatke iz baze
- pretvara ih u DTO objekte
- vraća JSON odgovor prema App projektu

Kontroler nije obična klasa samo po nazivu. Posebna je zato što koristi atribute koji ASP.NET Core aplikaciji govore da tu klasu treba tretirati kao web API endpoint.

### DTO

DTO znači <strong>Data Transfer Object</strong>. To je klasa koja služi za prijenos podataka između API-ja i App projekta.

DTO koristimo zato što:

- ne moramo slati cijeli entity model baze
- frontend dobiva samo ono što mu treba za prikaz
- kasnije lakše odvajamo modele za prikaz od modela za spremanje
- arhitektura ostaje čišća i preglednija

<InfoBox type="success" title="Jednostavno pravilo">
Entity model opisuje podatke u sustavu i bazi. DTO opisuje podatke koje šaljemo preko API-ja prema korisničkom sučelju.
</InfoBox>

- u adresnoj traci preglednika kad pokreneš API
- u izlazu pokretanja aplikacije u Visual Studio prozoru

### Što dodajemo u `Program.cs` i zašto

U App projektu dodaj ovu registraciju među ostale servise:

~~~csharp
builder.Services.AddScoped(sp => new HttpClient
{
    BaseAddress = new Uri("https://localhost:7220/")
});
~~~

Ovaj kod radi tri važne stvari:

- registrira <code>HttpClient</code> u DI sustav aplikacije
- postavlja baznu adresu API projekta
- omogućuje da na Razor stranici možeš napisati <code>@inject HttpClient Http</code> i zatim zvati rute poput <code>api/animals</code>

Drugim riječima, kad na stranici kasnije pozoveš:

~~~csharp
await Http.GetFromJsonAsync<List<AnimalDto>>("api/animals")
~~~

stvarna adresa postaje:

~~~text
https://localhost:7220/api/animals
~~~

<InfoBox type="warning" title="Važno">
Ako je API ugašen, ako port nije točan ili ako <code>HttpClient</code> nije registriran, App neće moći dohvatiti podatke.
</InfoBox>

using AzilEdu.Api.Data;
using AzilEdu.Shared.DTOs;
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Controllers;

[ApiController]
[Route("api/[controller]")]
public class AnimalsController : ControllerBase
{
    private readonly AzilEduDbContext _context;

    public AnimalsController(AzilEduDbContext context)
    {
        _context = context;
    }
}
~~~

### Što znači `[ApiController]`

Atribut <code>[ApiController]</code> govori ASP.NET Core aplikaciji da je ova klasa Web API kontroler.

To donosi nekoliko važnih ponašanja:

- klasa se uključuje u API pipeline
- parametri i body zahtjeva obrađuju se po pravilima za API
- framework automatski pomaže oko validacije i HTTP odgovora
- kontroler se koristi za rute koje vraćaju podatke, a ne HTML prikaz

### Što znači `[Route("api/[controller]")]`

Ovaj atribut definira osnovnu adresu kontrolera.

Dio <code>[controller]</code> automatski uzima naziv klase bez riječi <code>Controller</code>.

Zato:

- klasa <code>AnimalsController</code>
- postaje ruta <code>/api/animals</code>

Aplikacija dakle ne pogađa rutu nasumično. ASP.NET Core čita atribute i iz njih gradi adresu kojoj pristupaš preko HTTP-a.

### Korak 2 – GET metoda

Sada dodajemo metodu koja vraća listu životinja.

~~~csharp
[HttpGet]
public async Task<ActionResult<List<AnimalDto>>> GetAnimals()
{
    var animals = await _context.Animals
        .OrderBy(a => a.Name)
        .Select(a => new AnimalDto
        {
            Id = a.Id,
            Name = a.Name,
            Species = a.Species,
            Breed = a.Breed,
            Gender = a.Gender,
            Age = a.Age,
            ArrivalDate = a.ArrivalDate,
            IsAdopted = a.IsAdopted,
            ImageUrl = a.ImageUrl,
            Description = a.Description
        })
        .ToListAsync();

    return Ok(animals);
}
~~~

### Što točno znači `[HttpGet]`

Atribut <code>[HttpGet]</code> govori da se ova metoda pokreće kad stigne HTTP GET zahtjev.

GET koristimo kad:

- dohvaćamo podatke
- ne mijenjamo stanje baze
- želimo vratiti listu ili pojedini zapis

U ovoj lekciji GET metoda znači:

- App pošalje GET zahtjev na <code>/api/animals</code>
- ASP.NET Core prepozna da ruta pripada <code>AnimalsController</code>
- atribut <code>[HttpGet]</code> odabere metodu <code>GetAnimals()</code>
- metoda pročita podatke iz baze, pretvori ih u DTO i vrati ih kao JSON

### Cjeloviti sadržaj datoteke `AnimalsController.cs`

~~~csharp
using AzilEdu.Api.Data;

using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Controllers;

[ApiController]

public class AnimalsController : ControllerBase
{

    }

    [HttpGet]
    public async Task<ActionResult<List<AnimalDto>>> GetAnimals()
    {

            .OrderBy(a => a.Name)
            .Select(a => new AnimalDto
            {
                Id = a.Id,
                Name = a.Name,
                Species = a.Species,
                Breed = a.Breed,
                Gender = a.Gender,
                Age = a.Age,
                ArrivalDate = a.ArrivalDate,
                IsAdopted = a.IsAdopted,
                ImageUrl = a.ImageUrl,
                Description = a.Description
            })
            .ToListAsync();

        return Ok(animals);
    }
}
~~~

### Zašto kod `MudChip` pišemo `T="string"`

`MudChip` je generička komponenta. To znači da u pozadini radi s određenim tipom podatka.

U našem slučaju unutar komponente prikazujemo tekstualnu vrijednost, odnosno string. Razor iz child sadržaja ponekad ne može dovoljno sigurno zaključiti koji tip treba koristiti. Zato taj tip zadajemo eksplicitno:

~~~razor
<MudChip T="string">U azilu</MudChip>
~~~

Time ne "gasimo" samo grešku, nego jasno govorimo kompilatoru: ova instanca komponente radi s tekstualnim sadržajem.

### Korak 4 – `@code` dio i dohvat podataka

Sada dodajemo listu i metodu za učitavanje podataka.

~~~razor
@code {
    private List<AnimalDto> animals = new();

    protected override async Task OnInitializedAsync()
    {
        List<AnimalDto>? result = await Http.GetFromJsonAsync<List<AnimalDto>>("api/animals");
        animals = result ?? new List<AnimalDto>();
    }
}
~~~

### Zašto ovdje koristimo eksplicitnu varijablu `List<AnimalDto>? result`

U Razor datotekama ponekad je sigurnije eksplicitno napisati puni tip rezultata. Tako je kod čitljiviji, a smanjuje se mogućnost da editor ili parser krivo interpretira generički poziv.

Ako ti se ranije pojavljivalo upozorenje uz <code>GetFromJsonAsync&lt;List&lt;AnimalDto&gt;&gt;</code>, provjeri dvije stvari:

- nalazi li se kod stvarno unutar <code>@code { }</code> bloka
- imaš li dodan <code>@using System.Net.Http.Json</code>

## 12. Sažetak lekcije

U ovoj lekciji podaci putuju ovim redoslijedom:

1. u `AzilEdu.App` stranica `Animals.razor` koristi `HttpClient`
2. `HttpClient` šalje GET zahtjev na `https://localhost:7220/api/animals`
3. ASP.NET Core prepoznaje da ruta pripada klasi `AnimalsController`
4. atribut `[HttpGet]` usmjerava zahtjev na metodu `GetAnimals()`
5. kontroler koristi `AzilEduDbContext` i čita podatke iz tablice `Animals`
6. entity objekti se mapiraju u `AnimalDto` objekte
7. API vraća JSON odgovor prema App projektu
8. `GetFromJsonAsync` u `Animals.razor` pretvara JSON u `List<AnimalDto>`
9. Razor stranica tu listu prikazuje u `MudTable` komponenti

To je prvi puni primjer pravilnog odvajanja korisničkog sučelja od pristupa bazi. Na toj osnovi u sljedećoj lekciji nadograđujemo unos, uređivanje i brisanje podataka preko API-ja.
