## 1. Gdje smo stali i što sada nadograđujemo?

Do sada smo napravili nekoliko važnih koraka:

1. upoznali strukturu rješenja
2. složili osnovni izgled aplikacije i navigaciju
3. napravili stranicu <code>Animals.razor</code>
4. povezali prikaz s bazom i prikazali podatke iz tablice <code>Animals</code>

Sada na toj istoj osnovi dodajemo tri nove mogućnosti:

- unos nove životinje
- uređivanje postojeće životinje
- brisanje postojeće životinje

To je prirodan nastavak prethodne lekcije. Prvo smo naučili kako dohvatiti i prikazati podatke iz baze. Sada učimo kako nad tim istim podacima napraviti promjene.

## 2. Model `Animal` ostaje usklađen s prethodnom lekcijom

U ovoj lekciji koristi se isti model koji smo imali i ranije. Ne uvodimo novi ili pojednostavljeni model jer želimo da se sve što sada radimo nastavlja na već postojeći projekt.

```csharp
namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Species { get; set; } = string.Empty;
    public string Breed { get; set; } = string.Empty;
    public string Gender { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public bool IsAdopted { get; set; }
    public string ImageUrl { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
}
```

### Zašto je važno da model ostane isti?

Ako bi se u ovoj lekciji koristio drukčiji model nego u prethodnoj, odmah bi nastala zbrka:

- neka polja bi nestala iz prikaza
- forme ne bi više odgovarale stvarnim podacima
- bilo bi nejasno radi li se o istoj tablici ili o nekom novom primjeru

Zato sada ne mijenjamo temelj projekta, nego gradimo dalje na postojećem modelu.

## 3. Nadogradnja stranice `Animals.razor`

U prethodnoj lekciji složili smo stranicu koja dohvaća i prikazuje životinje iz baze. U ovoj lekciji tu istu stranicu nadograđujemo tako da:

- iznad tablice dodamo gumb za unos nove životinje
- u svaki red tablice dodamo gumbe <strong>Uredi</strong> i <strong>Obriši</strong>
- nakon brisanja odmah osvježimo prikaz

Važno je ne preskočiti logiku redoslijeda. Prvo ćemo pogledati koje dijelove dodajemo, zatim ćemo ih objasniti, a tek na kraju dobiti cjeloviti kod stranice.

### 3.1 Početak stranice i ovisnosti

Na početku stranice trebaju nam usings, injekcije i lista podataka.

```razor
@page "/animals"
@using Microsoft.EntityFrameworkCore
@using AzilEdu.Shared.Models
@using AzilEdu.Api.Data
@inject AzilEduDbContext Db
@inject NavigationManager Navigation

<PageTitle>Životinje</PageTitle>

@code {
    private List<Animal> animals = new();
}
```

### Što ovdje radimo?

- definiramo rutu stranice <code>/animals</code>
- uključujemo namespaceove koji su potrebni za bazu i model
- preko <code>@inject</code> dobivamo pristup bazi i navigaciji
- pripremamo listu <code>animals</code> u koju ćemo spremiti podatke iz baze

### 3.2 Dohvat podataka iz baze

Sada dodajemo metodu koja će se pozvati pri učitavanju stranice.

```razor
@code {
    private List<Animal> animals = new();

    protected override async Task OnInitializedAsync()
    {
        animals = await Db.Animals.ToListAsync();
    }
}
```

### Čemu to služi?

Ovdje se događa ključna promjena u odnosu na ranije probne primjere. Podaci više ne dolaze iz ručno upisane liste, nego iz stvarne tablice <code>Animals</code> u bazi.

Kad se stranica otvori:

- Blazor pozove <code>OnInitializedAsync()</code>
- preko <code>Db.Animals.ToListAsync()</code> pročita sve zapise
- rezultat spremi u listu <code>animals</code>
- tablica zatim prikazuje stvarne podatke

### 3.3 Naslov i gumb za dodavanje nove životinje

Iznad tablice dodajemo naslov i gumb koji vodi na stranicu za unos.


### Zašto je ovo važno?

Stranica za prikaz više nije samo pasivna lista. Gumb iznad tablice pretvara je u polaznu točku za daljnji rad. S te iste stranice kreće unos novog zapisa.

A u <code>@code</code> dijelu dodajemo pripadajuće metode.

```razor
private void EditAnimal(int id)
{
    Navigation.NavigateTo($"/animals/edit/{id}");
}

private async Task DeleteAnimal(int id)
{
    var animal = await Db.Animals.FindAsync(id);

    if (animal is not null)
    {
        Db.Animals.Remove(animal);
        await Db.SaveChangesAsync();
        animals = await Db.Animals.ToListAsync();
    }
}
```

### Zašto nakon brisanja ponovno učitavamo listu?

Nakon brisanja želimo odmah vidjeti novo stanje tablice. Zato ponovno dohvaćamo podatke iz baze. Tako prikaz ostaje usklađen sa stvarnim sadržajem tablice <code>Animals</code>.

## 4. Stranica za unos nove životinje

Sada dodajemo stranicu za unos nove životinje. I ovdje ćemo ići postupno: najprije ruta i osnovni objekt, zatim forma, zatim spremanje, a tek na kraju kompletan kod.

Primjer putanje datoteke:

```text
Pages/Animals/AnimalCreate.razor
```

```razor

@using AzilEdu.Api.Data
```

- definiramo rutu stranice za unos

- datum vežemo na pomoćnu varijablu <code>arrivalDate</code>

## 5. Stranica za uređivanje postojeće životinje

Stranica za uređivanje jako je slična stranici za unos, ali postoji jedna važna razlika: ne stvaramo novi objekt, nego učitavamo postojeći zapis iz baze i mijenjamo ga.

Primjer putanje datoteke:

```text
Pages/Animals/AnimalEdit.razor
```

### 5.1 Ruta i dohvat zapisa po `Id`

```razor
@page "/animals/edit/{Id:int}"
@using AzilEdu.Shared.Models
@using AzilEdu.Api.Data
@inject AzilEduDbContext Db
@inject NavigationManager Navigation
@inject ISnackbar Snackbar

@code {
    [Parameter]
    public int Id { get; set; }

    private Animal? animal;
    private DateTime? arrivalDate;

    protected override async Task OnInitializedAsync()
    {
        animal = await Db.Animals.FindAsync(Id);

        if (animal is not null)
        {
            arrivalDate = animal.ArrivalDate;
        }
    }
}
```

### Što je ovdje bitno razumjeti?

- <code>Id</code> dolazi iz rute
- pomoću tog identifikatora tražimo točno jedan zapis u tablici
- forma se puni postojećim vrijednostima jer radimo nad pronađenim objektom

## 6. Brisanje zapisa kao završni korak CRUD logike

Brisanje je već ugrađeno u stranicu <code>Animals.razor</code>, ali vrijedi ga još jednom jasno izdvojiti jer zatvara puni CRUD krug.

```csharp
private async Task DeleteAnimal(int id)
{
    var animal = await Db.Animals.FindAsync(id);

    if (animal is not null)
    {
        Db.Animals.Remove(animal);
        await Db.SaveChangesAsync();
        animals = await Db.Animals.ToListAsync();
    }
}
```

### Što se ovdje događa?

1. pronalazi se točno onaj zapis koji želimo obrisati
2. ako zapis postoji, predaje se metodi <code>Remove()</code>
3. poziv <code>SaveChangesAsync()</code> stvarno briše red iz baze
4. nakon toga se lista ponovno učitava kako bi prikaz odmah bio ažuran
