## Ishodi ove lekcije

Nakon ove lekcije učenik bi trebao moći:

- objasniti razliku između **pretrage**, **filtriranja** i **sortiranja**
- dodati **search** polje iznad postojeće MudTable tablice
- povezati **MudTable** s logikom koja filtrira retke
- omogućiti sortiranje po važnim stupcima
- dodati jednostavne filtere, npr. po vrsti životinje i statusu udomljenja
- razumjeti što MudTable nudi ugrađeno, a što mora napisati sam

## Što već imamo prije početka?

Do ove lekcije projekt već sadrži:

- model <code>Animal</code>
- tablicu <code>Animals</code> u bazi
- EF Core i migracije
- prikaz popisa životinja iz baze
- unos, uređivanje i brisanje životinja
- osnovni servis ili logiku za dohvat podataka

To znači da se u ovoj lekciji **ne vraćamo na bazu od nule**, nego nadograđujemo postojeću stranicu za prikaz životinja.

## Što MudTable može sama, a što ipak moramo napisati?

MudTable podržava tablični prikaz, paginaciju, sortiranje i filtriranje, ali ne znači da će aplikacija sama pogoditi kako ti želiš da korisnik pretražuje podatke. Komponenta daje mogućnosti, ali logiku pretrage i filtera često i dalje definira programer.

To praktično znači:

- **sortiranje** možemo relativno uredno povezati kroz zaglavlja stupaca
- **pretragu** obično moramo vezati uz <code>MudTextField</code> i vlastitu filter funkciju
- **posebne filtere** poput "prikaži samo pse" ili "prikaži samo neudomljene" također moramo sami osmisliti

<InfoBox type="success" title="Glavna ideja lekcije">
Ne učimo učenike da komponenta "magijom" sve rješava sama. Učimo ih kako iskoristiti mogućnosti komponente i povezati ih s vlastitim podacima i poslovnom logikom.
</InfoBox>

## Korak 1 — Priprema stranice za rad s pretragom i filterima

Na postojećoj stranici s popisom životinja dodajemo nekoliko novih svojstava koja će upravljati prikazom tablice.

Prvo definiramo polja za:

- tekst pretrage
- odabranu vrstu životinje
- status udomljenja

Kod može izgledati ovako:

```razor
@code {
    private string searchString = string.Empty;
    private string? selectedSpecies;
    private bool? adoptedFilter = null;
}
```

### Objašnjenje

- <code>searchString</code> sadrži tekst koji korisnik upisuje u tražilicu
- <code>selectedSpecies</code> služi za izbor vrste, npr. pas, mačka ili ptica
- <code>adoptedFilter</code> je nullable <code>bool</code>, što znači da može imati tri stanja:
  - <code>true</code> → prikazuj samo udomljene
  - <code>false</code> → prikazuj samo neudomljene
  - <code>null</code> → ne filtriraj po statusu

To treće stanje je važno jer korisnik ponekad želi vidjeti sve zapise.

## Korak 3 — Pisanje filter funkcije

Sada trebamo definirati logiku koja će odlučiti prikazuje li se pojedini redak u tablici ili ne.

Kod može izgledati ovako:

```csharp
private bool FilterAnimals(Animal animal)
{
    if (!string.IsNullOrWhiteSpace(searchString))
    {
        var term = searchString.Trim().ToLower();

        var matchesSearch =
            (!string.IsNullOrWhiteSpace(animal.Name) && animal.Name.ToLower().Contains(term)) ||
            (!string.IsNullOrWhiteSpace(animal.Breed) && animal.Breed.ToLower().Contains(term)) ||
            (!string.IsNullOrWhiteSpace(animal.Description) && animal.Description.ToLower().Contains(term));

        if (!matchesSearch)
            return false;
    }

    if (!string.IsNullOrWhiteSpace(selectedSpecies))
    {
        if (!string.Equals(animal.Species, selectedSpecies, StringComparison.OrdinalIgnoreCase))
            return false;
    }

    if (adoptedFilter.HasValue)
    {
        if (animal.IsAdopted != adoptedFilter.Value)
            return false;
    }

    return true;
}
```

### Objašnjenje logike

Ova metoda prima jedan objekt <code>Animal</code> i vraća:

- <code>true</code> ako taj zapis treba ostati vidljiv u tablici
- <code>false</code> ako ga treba sakriti

Logika radi ovim redoslijedom:

1. ako korisnik nije upisao ništa u tražilicu, taj dio se preskače
2. ako jest, provjerava se sadrži li ime, pasmina ili opis traženi pojam
3. ako je odabrana vrsta životinje, provjerava se podudara li se zapis s tom vrstom
4. ako je odabran status udomljenja, provjerava se podudara li se i taj uvjet
5. samo ako zapis prođe sve uvjete, ostaje u tablici

<InfoBox type="warning" title="Važna didaktička napomena">
Učenici često napišu jedan veliki uvjet u jednoj liniji. To tehnički može raditi, ali je vrlo teško za čitanje i održavanje. U ovoj lekciji je važnije da logika bude jasna nego da bude što kraća.
</InfoBox>

### Što je ovdje novo?

Najvažnije nove stvari su:

- <code>Filter="FilterAnimals"</code> povezuje našu metodu s tablicom
- <code>MudTableSortLabel</code> omogućuje sortiranje po odabranim stupcima
- korisnik sada može i pretraživati i sortirati istu tablicu

### Što nije automatsko?

Iako MudTable podržava filtriranje i sortiranje, ona **ne zna sama**:

- po kojim poljima želiš pretraživati
- koji su poslovno važni filteri
- treba li opis uključiti u pretragu
- kako prikazati status udomljenja

To sve i dalje definira programer.

## Korak 5 — Učitavanje podataka iz baze

Ako smo u prethodnim lekcijama već prebacili listu životinja na bazu, tada se ovdje samo naslanjamo na postojeći dohvat podataka.

Primjer:

```csharp
private List<Animal> animals = new();

protected override async Task OnInitializedAsync()
{
    animals = await AnimalService.GetAllAsync();
}
```

Ako servis još nije uveden, ista logika može privremeno stajati i direktno u stranici, ali cilj je da projekt ostane uredan i da rad s bazom bude izdvojen iz UI sloja.

<InfoBox type="info" title="Važno za razumijevanje">
Pretraga i filter u ovoj lekciji rade nad listom koja je već učitana u memoriju. To je dobar i razumljiv početni korak. Kasnije, kod većih količina podataka, dio te logike može se seliti prema bazi i upitima nad bazom.
</InfoBox>

## Najčešće greške učenika

<StepGrid>
  <StepCard title="1. Filter funkcija nije povezana s tablicom">
    Učenik napiše metodu <code>FilterAnimals</code>, ali je ne doda u <code>Filter</code> atribut tablice.
  </StepCard>
  <StepCard title="2. Pretraga nije Immediate">
    Učenik očekuje da se rezultati mijenjaju odmah, ali bez <code>Immediate="true"</code> ponašanje nije onakvo kakvo očekuje.
  </StepCard>
  <StepCard title="3. Pogrešni podaci u filteru vrste">
    Ako u bazi piše <code>pas</code>, a u filtru traži <code>Pas</code>, bez pravilne usporedbe može doći do problema.
  </StepCard>
  <StepCard title="4. Prazna tablica bez poruke">
    Ako nema <code>NoRecordsContent</code>, korisnik ne zna je li problem u filteru ili u aplikaciji.
  </StepCard>
</StepGrid>

## Vježba na satu

1. Dodaj tekstualnu tražilicu koja pretražuje barem ime i pasminu životinje.  
2. Dodaj filter po vrsti životinje.  
3. Dodaj filter po statusu udomljenja.  
4. Omogući sortiranje po imenu i datumu dolaska.  
5. Dodaj gumb za resetiranje svih filtera.  
6. Prikaži poruku ako pretraga ne vrati nijedan rezultat.

## Domaći zadatak

Nadogradi postojeću stranicu tako da korisnik može:

- pretraživati po najmanje tri polja
- filtrirati po vrsti životinje
- filtrirati po statusu udomljenja
- sortirati po najmanje tri stupca
- vidjeti poruku kada nema rezultata

Dodatni izazov:

- dodaj filter koji prikazuje samo životinje mlađe od određene dobi
- ili dodaj sortiranje po statusu udomljenja
