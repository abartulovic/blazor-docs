## Zašto odmah radimo odvojene projekte?

Najčešća početnička greška je da se sve trpa u jedan projekt: UI, modeli, API logika, pomoćne klase i nasumične datoteke. To kratko djeluje brže, ali kasnije postane nered.

<CardGrid>
  <Card title="AzilEdu.App" accent="#2563eb">
    Blazor korisničko sučelje. Ovdje su stranice, navigacija, forme i prikaz
    podataka koje korisnik vidi u pregledniku.
  </Card>
  <Card title="AzilEdu.Api" accent="#10b981">
    Backend sloj. Prima HTTP zahtjeve, obrađuje ih i kasnije komunicira s bazom
    podataka.
  </Card>
  <Card title="AzilEdu.Shared" accent="#a855f7">
    Zajednička biblioteka. Ovdje stavljamo DTO modele i tipove koje koriste i
    App i Api.
  </Card>
</CardGrid>

```text
Korisnik -> AzilEdu.App -> AzilEdu.Api -> baza/podaci -> odgovor -> AzilEdu.App
```

<Note title="Ovo mora ostati kristalno jasno" tone="blue">
  <strong>App nije API.</strong>
  <br />
  <strong>API nije baza.</strong>
  <br />
  <strong>Shared nije mjesto za Razor stranice ni kontrolere.</strong>
</Note>

### Primjer početnog sadržaja za `Home.razor`

```razor
@page "/"

<PageTitle>Azil za životinje</PageTitle>

<h1>Dobro došli!</h1>

<p>
    AzilEdu je aplikacija za upravljanje podacima o životinjama,
    vrstama, pasminama i udomljavanju.
</p>

<p>
    U sljedećim lekcijama postupno ćemo dodavati nove funkcionalnosti
    i povezivati korisničko sučelje s API slojem.
</p>
```

<Note title="Što ovaj kod radi" tone="blue">
  <ul>
    <li>
      <code>@page "/"</code> definira početnu rutu stranice
    </li>
    <li>
      <code>&lt;PageTitle&gt;</code> postavlja naslov kartice u pregledniku
    </li>
    <li>
      <code>&lt;h1&gt;</code> i <code>&lt;p&gt;</code> daju jasan početni
      sadržaj stranice
    </li>
  </ul>
