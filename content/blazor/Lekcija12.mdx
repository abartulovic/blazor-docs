## Cilj predavanja

U ovoj lekciji želimo omogućiti da sustav zna:

- je li životinja udomljena
- kada je udomljena
- tko je udomitelj
- koji su kontakt podaci udomitelja
- postoji li napomena uz udomljenje
- kako nakon udomljenja automatski promijeniti status životinje

Na kraju lekcije jedna životinja može imati svoj zapis o udomljenju, a aplikacija će moći razlikovati dostupne životinje od već udomljenih.

---

## Poslovna logika prije koda

Prije nego napišemo ijedan redak koda, moramo jasno razumjeti pravila sustava.

### Osnovno pravilo

Jedna životinja može imati najviše jedno aktivno udomljenje.

To znači:
- ne želimo dva paralelna zapisa o udomljenju za istu životinju
- kada se životinja udomi, njezin status više ne bi smio biti "Dostupna"
- detalji životinje trebaju jasno pokazivati postoji li zapis o udomljenju

<InfoBox type="warning" title="Ovdje prvi put uvodimo ozbiljniju poslovnu logiku">
  Ovo više nije samo CRUD. Sada postoji pravilo sustava: kada nastane udomljenje, treba se promijeniti i stanje životinje. Upravo tu aplikacija prelazi iz školskog primjera u ozbiljniji poslovni projekt.
</InfoBox>

## Novi entitet: `Adoption`

### Predložena klasa

```csharp
public class Adoption
{
    public int Id { get; set; }

    public int AnimalId { get; set; }
    public Animal? Animal { get; set; }

    public DateTime AdoptionDate { get; set; } = DateTime.Today;

    public string AdopterFullName { get; set; } = string.Empty;
    public string PhoneNumber { get; set; } = string.Empty;
    public string? Email { get; set; }
    public string? Address { get; set; }
    public string? Note { get; set; }
}
```

### Objašnjenje polja

- `AnimalId` povezuje udomljenje s točno jednom životinjom
- `AdoptionDate` označava datum udomljenja
- `AdopterFullName` čuva ime i prezime udomitelja
- `PhoneNumber` je važan za kontakt
- `Email` i `Address` su dodatna polja koja mogu pomoći u evidenciji
- `Note` služi za dodatne napomene, npr. dogovor o kasnijoj kontroli

## Povezivanje s klasom `Animal`

U klasu `Animal` dodaj navigacijsko svojstvo:

```csharp
public Adoption? Adoption { get; set; }
```

### Zašto je ovdje veza jedan-prema-jedan?

U ovoj fazi projekta pretpostavljamo:
- jedna životinja ima najviše jedno udomljenje
- kada se udomi, to je završni zapis za taj poslovni proces

Kasnije bi se sustav mogao dodatno proširiti, ali za ovu aplikaciju ovo je sasvim dovoljno i didaktički čisto.

## Fluent API konfiguracija odnosa

Kako bismo jasno definirali odnos jedan-prema-jedan između `Animal` i `Adoption`, dobro je u `OnModelCreating` dodati konfiguraciju:

```csharp
modelBuilder.Entity<Animal>()
    .HasOne(a => a.Adoption)
    .WithOne(a => a.Animal)
    .HasForeignKey<Adoption>(a => a.AnimalId)
    .OnDelete(DeleteBehavior.Cascade);
```

### Zašto je to korisno?

Jer učvršćuješ pravilo koje već poslovno želiš:
- jedan zapis o udomljenju pripada jednoj životinji
- relacija je eksplicitna i jasna

## Servis za rad s udomljenjima

### Sučelje servisa

```csharp
public interface IAdoptionService
{
    Task<Adoption?> GetByAnimalIdAsync(int animalId);
    Task<bool> CreateAsync(Adoption adoption);
    Task DeleteAsync(int adoptionId);
}
```

### Implementacija servisa

```csharp
public class AdoptionService : IAdoptionService
{
    private readonly AzilEduDbContext _db;

    public AdoptionService(AzilEduDbContext db)
    {
        _db = db;
    }

    public async Task<Adoption?> GetByAnimalIdAsync(int animalId)
    {
        return await _db.Adoptions
            .FirstOrDefaultAsync(a => a.AnimalId == animalId);
    }

    public async Task<bool> CreateAsync(Adoption adoption)
    {
        var exists = await _db.Adoptions.AnyAsync(a => a.AnimalId == adoption.AnimalId);
        if (exists)
            return false;

        _db.Adoptions.Add(adoption);
        await _db.SaveChangesAsync();
        return true;
    }

    public async Task DeleteAsync(int adoptionId)
    {
        var entity = await _db.Adoptions.FindAsync(adoptionId);
        if (entity is null)
            return;

        _db.Adoptions.Remove(entity);
        await _db.SaveChangesAsync();
    }
}
```

### Registracija servisa

```csharp
builder.Services.AddScoped<IAdoptionService, AdoptionService>();
```

## Važno proširenje: promjena statusa životinje nakon udomljenja

Ovo je najvažniji dio lekcije. Kada korisnik stvori zapis o udomljenju, status životinje treba automatski postati npr. **Udomljena**.

### Zašto to radimo?

Zato što podaci moraju ostati dosljedni. Ne želimo situaciju da:
- životinja ima zapis o udomljenju
- ali u tablici i dalje izgleda kao da je dostupna za udomljenje

To bi bio loš podatkovni model i loše korisničko iskustvo.

---

## Nadogradnja servisa poslovnom logikom

Pretpostavimo da već postoji status s nazivom `Udomljena`.

### Primjer metode `CreateAsync` s promjenom statusa

```csharp
public async Task<bool> CreateAsync(Adoption adoption)
{
    var exists = await _db.Adoptions.AnyAsync(a => a.AnimalId == adoption.AnimalId);
    if (exists)
        return false;

    var animal = await _db.Animals.FirstOrDefaultAsync(a => a.Id == adoption.AnimalId);
    if (animal is null)
        return false;

    var adoptedStatus = await _db.AnimalStatuses
        .FirstOrDefaultAsync(s => s.Name == "Udomljena");

    if (adoptedStatus is null)
        return false;

    animal.AnimalStatusId = adoptedStatus.Id;

    _db.Adoptions.Add(adoption);
    await _db.SaveChangesAsync();
    return true;
}
```

<InfoBox type="success" title="Što učenici ovdje uče?">
  Uče da servis ne služi samo za spremanje podataka, nego i za provođenje poslovnih pravila aplikacije.
</InfoBox>

## Forma za novo udomljenje

Najčišći pristup je napraviti zasebnu stranicu, npr.:

```text
/animals/{id}/adopt
```

### Početak Razor stranice

```razor
@page "/animals/{AnimalId:int}/adopt"
@inject IAdoptionService AdoptionService
@inject NavigationManager NavigationManager
@inject ISnackbar Snackbar

<PageTitle>Udomljenje životinje</PageTitle>
```

### Kod-behind logika

```razor
@code {
    [Parameter]
    public int AnimalId { get; set; }

    private Adoption model = new();

    protected override void OnInitialized()
    {
        model.AnimalId = AnimalId;
        model.AdoptionDate = DateTime.Today;
    }

    private async Task SaveAsync()
    {
        var success = await AdoptionService.CreateAsync(model);

        if (!success)
        {
            Snackbar.Add("Udomljenje nije moguće spremiti. Provjeri postoji li već zapis ili status.", Severity.Error);
            return;
        }

        Snackbar.Add("Udomljenje je uspješno spremljeno.", Severity.Success);
        NavigationManager.NavigateTo($"/animals/{AnimalId}/details");
    }
}
```

## DataAnnotations validacija

Kako bi unos bio kvalitetniji, model možeš proširiti validacijskim atributima.

```csharp
using System.ComponentModel.DataAnnotations;

public class Adoption
{
    public int Id { get; set; }

    public int AnimalId { get; set; }
    public Animal? Animal { get; set; }

    [Required]
    public DateTime AdoptionDate { get; set; } = DateTime.Today;

    [Required]
    [StringLength(100)]
    public string AdopterFullName { get; set; } = string.Empty;

    [Required]
    [StringLength(30)]
    public string PhoneNumber { get; set; } = string.Empty;

    [StringLength(100)]
    [EmailAddress]
    public string? Email { get; set; }

    [StringLength(200)]
    public string? Address { get; set; }

    [StringLength(1000)]
    public string? Note { get; set; }
}
```

## Brisanje ili poništavanje udomljenja

Ovdje treba biti oprezan. Ako izbrišeš zapis o udomljenju, trebaš odlučiti što će se dogoditi sa statusom životinje.

### Moguća pravila

1. kada se udomljenje obriše, status se vrati na `Dostupna`
2. brisanje nije dopušteno bez dodatne potvrde administratora
3. umjesto brisanja uvodi se “storno udomljenja”

Za učenike je u ovoj fazi dovoljno pravilo broj 1.

### Primjer logike za brisanje

```csharp
public async Task DeleteAsync(int adoptionId)
{
    var adoption = await _db.Adoptions.FirstOrDefaultAsync(a => a.Id == adoptionId);
    if (adoption is null)
        return;

    var animal = await _db.Animals.FirstOrDefaultAsync(a => a.Id == adoption.AnimalId);
    var availableStatus = await _db.AnimalStatuses.FirstOrDefaultAsync(s => s.Name == "Dostupna");

    if (animal is not null && availableStatus is not null)
    {
        animal.AnimalStatusId = availableStatus.Id;
    }

    _db.Adoptions.Remove(adoption);
    await _db.SaveChangesAsync();
}
```

<InfoBox type="warning" title="Ovo je poslovna odluka, ne samo tehnički detalj">
  U stvarnoj aplikaciji moraš odlučiti što znači brisanje udomljenja. Ovdje biramo najjednostavniju varijantu da bi učenici razumjeli tok podataka i poslovno pravilo.
</InfoBox>
