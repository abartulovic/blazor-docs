U prethodnoj lekciji omogućili smo rad nad stvarnim podacima iz baze i pokazali **create, edit i delete** nad tablicom <code>Animals</code>.

Sada ne dodajemo novu veliku funkcionalnost, nego postojeći kod činimo boljim:

- izdvajamo rad s bazom u **servis**
- uvodimo **potvrdu brisanja**
- pojačavamo **validaciju forme**
- pojednostavljujemo stranice tako da budu lakše za čitanje i održavanje
</InfoBox>

<InfoBox type="success" title="Zašto je ovo važno?">
Ako sav kod ostane unutar <code>.razor</code> stranica, projekt brzo postaje nepregledan. U ozbiljnijim aplikacijama logika pristupa podacima ne ostaje razbacana po UI-u, nego se organizira u zasebne klase i servise.
</InfoBox>

## Ishodi učenja

Na kraju ove lekcije učenik bi trebao moći:

- objasniti zašto CRUD logiku nije dobro držati samo u stranici
- izraditi servis <code>AnimalService</code>
- kroz servis dohvaćati, dodavati, uređivati i brisati zapise
- dodati potvrdu prije brisanja zapisa
- poboljšati validaciju forme za unos i uređivanje životinje
- prepoznati razliku između UI logike i logike rada s bazom

## Podsjetnik: gdje smo stali?

U projektu već imamo:

- model <code>Animal</code>
- bazu i migracije
- tablicu <code>Animals</code>
- prikaz zapisa iz baze
- formu za dodavanje i uređivanje
- gumb za brisanje zapisa

Problem je što se u ovakvoj fazi često dogodi sljedeće:

- dohvat podataka stoji u stranici
- spremanje podataka stoji u stranici
- brisanje stoji u stranici
- validacija je djelomična
- brisanje se izvršava odmah, bez potvrde

Takav kod radi, ali nije uredan.

## 2. Gdje ćemo smjestiti servis?

Za ovu fazu projekta sasvim je dovoljno napraviti servis unutar **Api** projekta, u folderu npr.:

```text
AzilEdu.Api/Services/AnimalService.cs
```

Ako želiš, kasnije možeš dodati i sučelje:

```text

```

U ovoj lekciji možeš i bez sučelja ako želiš pojednostaviti gradivo. Važnije je da učenici razumiju ideju izdvajanja logike nego da odmah uvode dodatnu apstrakciju.

## 3. Izrada servisa AnimalService

Napravimo novu klasu <code>AnimalService</code>.

```csharp
using AzilEdu.Api.Data;
using AzilEdu.Shared.Models;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Services;

public class AnimalService
{
    private readonly AzilEduDbContext _context;

    public AnimalService(AzilEduDbContext context)
    {
        _context = context;
    }

    public async Task<List<Animal>> GetAllAsync()
    {
        return await _context.Animals
            .OrderBy(a => a.Name)
            .ToListAsync();
    }

    public async Task<Animal?> GetByIdAsync(int id)
    {
        return await _context.Animals.FirstOrDefaultAsync(a => a.Id == id);
    }

    public async Task AddAsync(Animal animal)
    {
        _context.Animals.Add(animal);
        await _context.SaveChangesAsync();
    }

    public async Task UpdateAsync(Animal animal)
    {
        _context.Animals.Update(animal);
        await _context.SaveChangesAsync();
    }

    public async Task DeleteAsync(int id)
    {
        var animal = await _context.Animals.FindAsync(id);

        if (animal is null)
            return;

        _context.Animals.Remove(animal);
        await _context.SaveChangesAsync();
    }
}
```

<InfoBox type="info" title="Što ovdje učenici trebaju razumjeti?">
Nije cilj da samo prepišu metode. Moraju razumjeti da je servis mjesto gdje centraliziramo rad s bazom. Stranica tada samo zatraži podatke ili pošalje zahtjev servisu.
</InfoBox>

## 4. Registracija servisa u Program.cs

Da bi aplikacija mogla koristiti servis, treba ga registrirati u DI container.

U datoteci <code>Program.cs</code> dodaj:

```csharp
using AzilEdu.Api.Services;
```

Zatim registriraj servis:

```csharp
builder.Services.AddScoped<AnimalService>();
```

Idealno mjesto je uz ostale servise, nakon registracije <code>DbContext</code>-a.

```csharp
builder.Services.AddDbContext<AzilEduDbContext>(options =>
    options.UseSqlite(builder.Configuration.GetConnectionString("DefaultConnection")));

builder.Services.AddScoped<AnimalService>();
```

Pretpostavimo da već imamo stranicu za prikaz svih životinja, npr. <code>Animals.razor</code>.

## 6. Potvrda brisanja prije uklanjanja zapisa

Brisanje bez potvrde je loša praksa. Dovoljno je jedan pogrešan klik i zapis nestaje.

Za potvrdu možemo koristiti MudBlazor dialog. U ovoj lekciji pokazat ćemo jednostavnu varijantu.

### 6.1 Injektiranje dialog servisa

Na vrh stranice dodaj:

```razor
@inject IDialogService DialogService
```

### 6.2 Izrada jednostavne komponente za potvrdu brisanja

Napravimo komponentu npr.:

```text
Components/DeleteAnimalDialog.razor
```

Sadržaj:

```razor
@inject MudDialogInstance MudDialog

<MudDialog>
    <DialogContent>
        <MudText Typo="Typo.h6">Potvrda brisanja</MudText>
        <MudText Class="mt-2">
            Jeste li sigurni da želite obrisati životinju <strong>@AnimalName</strong>?
        </MudText>
    </DialogContent>
    <DialogActions>
        <MudButton Variant="Variant.Text" OnClick="Cancel">Odustani</MudButton>
        <MudButton Variant="Variant.Filled" Color="Color.Error" OnClick="Confirm">
            Obriši
        </MudButton>
    </DialogActions>
</MudDialog>

@code {
    [Parameter] public string AnimalName { get; set; } = string.Empty;

    private void Cancel() => MudDialog.Cancel();

    private void Confirm() => MudDialog.Close(DialogResult.Ok(true));
}
```

### 6.3 Pozivanje dijaloga iz stranice

U stranici za prikaz životinja dopuni metodu za brisanje:

```razor
private async Task OpenDeleteDialog(Animal animal)
{
    var parameters = new DialogParameters
    {
        { nameof(DeleteAnimalDialog.AnimalName), animal.Name }
    };

    var options = new DialogOptions
    {
        CloseOnEscapeKey = true,
        MaxWidth = MaxWidth.ExtraSmall,
        FullWidth = true
    };

    var dialog = await DialogService.ShowAsync<DeleteAnimalDialog>(
        "Potvrda brisanja",
        parameters,
        options);

    var result = await dialog.Result;

    if (!result.Canceled)
    {
        await AnimalService.DeleteAsync(animal.Id);
        Snackbar.Add("Životinja je uspješno obrisana.", Severity.Success);
        await LoadAnimalsAsync();
    }
}
```

## 7. Poboljšanje validacije forme

U prethodnoj lekciji već smo imali formu za unos ili uređivanje životinje. Sada je činimo ozbiljnijom.

### 7.1 Primjer modela s DataAnnotations atributima

Ako model još nema validaciju, možeš ga proširiti ovako:

```csharp
using System.ComponentModel.DataAnnotations;

namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }

    [Required(ErrorMessage = "Ime je obavezno.")]
    [StringLength(50, ErrorMessage = "Ime može imati najviše 50 znakova.")]
    public string Name { get; set; } = string.Empty;

    [Required(ErrorMessage = "Vrsta je obavezna.")]
    [StringLength(30, ErrorMessage = "Vrsta može imati najviše 30 znakova.")]
    public string Species { get; set; } = string.Empty;

    [StringLength(50, ErrorMessage = "Pasmina može imati najviše 50 znakova.")]
    public string? Breed { get; set; }

    [Range(0, 30, ErrorMessage = "Dob mora biti između 0 i 30 godina.")]
    public int Age { get; set; }

    [Required(ErrorMessage = "Datum dolaska je obavezan.")]
    public DateTime ArrivalDate { get; set; } = DateTime.Today;

    public bool IsAdopted { get; set; }
}
```

<InfoBox type="info" title="Zašto validacija ide i u model?">
Zato što pravila o podacima ne bi smjela živjeti samo u vizualnom dijelu forme. Model definira što je valjan podatak, a UI to pravilo samo prikazuje korisniku.
</InfoBox>

### 7.3 Uređivanje postojeće životinje preko servisa

Kod uređivanja je logika vrlo slična, ali prvo dohvaćamo postojeći zapis.

```razor
@page "/animals/edit/{id:int}"
@using AzilEdu.Api.Services
@using AzilEdu.Shared.Models
@inject AnimalService AnimalService
@inject NavigationManager NavigationManager
@inject ISnackbar Snackbar

<EditForm Model="animal" OnValidSubmit="UpdateAnimalAsync">
    <DataAnnotationsValidator />
    <ValidationSummary />

    <!-- Ovdje ide ista ili vrlo slična forma kao kod dodavanja -->
</EditForm>

@code {
    [Parameter] public int Id { get; set; }

    private Animal animal = new();

    protected override async Task OnInitializedAsync()
    {
        var result = await AnimalService.GetByIdAsync(Id);

        if (result is null)
        {
            Snackbar.Add("Tražena životinja nije pronađena.", Severity.Error);
            NavigationManager.NavigateTo("/animals");
            return;
        }

        animal = result;
    }

    private async Task UpdateAnimalAsync()
    {
        await AnimalService.UpdateAsync(animal);
        Snackbar.Add("Podaci o životinji su uspješno ažurirani.", Severity.Success);
        NavigationManager.NavigateTo("/animals");
    }
}
```
