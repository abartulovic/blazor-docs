## Ishodi učenja

Nakon ove lekcije učenik će moći:

- objasniti zašto details stranica nije isto što i edit forma
- dohvatiti jedan zapis po ID-u iz baze
- koristiti `Include` za dohvat povezanih podataka
- izraditi Blazor stranicu za prikaz detalja jedne životinje
- uredno prikazati podatke kroz MudBlazor kartice, grid i oznake statusa
- dodati navigaciju s popisa životinja prema detaljima

## 3. Podsjetnik na model nakon prošle lekcije

Nakon 8. predavanja model `Animal` više nije jednostavna tablica s tekstualnim poljima za sve, nego sadrži strane ključeve i navigacijska svojstva.

Primjer modela može izgledati ovako:

```csharp
public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Gender { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public string Description { get; set; } = string.Empty;
    public string? ChipNumber { get; set; }

    public int SpeciesId { get; set; }
    public Species? Species { get; set; }

    public int? BreedId { get; set; }
    public Breed? Breed { get; set; }

    public int AnimalStatusId { get; set; }
    public AnimalStatus? AnimalStatus { get; set; }
}
```

<InfoBox type="info" title="Što je ovdje novo?">
Sada više ne prikazujemo samo obične tekstualne vrijednosti. `SpeciesId`, `BreedId` i `AnimalStatusId` pokazuju na druge tablice, a `Species`, `Breed` i `AnimalStatus` predstavljaju povezana navigacijska svojstva.
</InfoBox>

## 4. Servis mora znati dohvatiti jednu životinju s povezanim podacima

Ako već imamo `AnimalService`, sada ga proširujemo tako da može dohvatiti <strong>jednu životinju po ID-u</strong> zajedno s relacijama.

Upravo je to srce ove lekcije. Ako servis ne vrati povezane podatke, details stranica neće imati što prikazati.

### Primjer metode u servisu

```csharp
using Microsoft.EntityFrameworkCore;

public class AnimalService
{
    private readonly AzilEduDbContext _context;

    public AnimalService(AzilEduDbContext context)
    {
        _context = context;
    }

    public async Task<Animal?> GetAnimalByIdAsync(int id)
    {
        return await _context.Animals
            .Include(a => a.Species)
            .Include(a => a.Breed)
            .Include(a => a.AnimalStatus)
            .FirstOrDefaultAsync(a => a.Id == id);
    }
}
```

<InfoBox type="warning" title="Zašto koristimo Include?">
Bez `Include` naredbi EF Core može vratiti samo osnovni zapis iz tablice `Animals`. To znači da bi `Species`, `Breed` i `AnimalStatus` mogli ostati prazni. Zato u details prikazu izričito tražimo i povezane podatke.
</InfoBox>

## 5. Izrada stranice `AnimalDetails.razor`

Sada izrađujemo novu stranicu koja prima ID životinje kroz URL.

Primjer početka stranice:

```razor
@page "/animals/details/{Id:int}"
@inject AnimalService AnimalService
@inject NavigationManager NavigationManager

<PageTitle>Detalji životinje</PageTitle>

```

Ovdje radimo tri važne stvari:

1. definiramo rutu koja prima `Id`
2. ubacujemo servis koji dohvaća životinju
3. pripremamo stranicu za prikaz jednog zapisa

### Parametar stranice i lokalne varijable

```razor
@code {
    [Parameter]
    public int Id { get; set; }

    private Animal? animal;
    private bool isLoading = true;
}
```

<InfoBox type="info" title="Zašto koristimo nullable Animal?">
`animal` je na početku `null` jer se podaci još nisu dohvatili iz baze. Tek nakon izvršavanja servisa dobit ćemo stvarni objekt ili zaključiti da zapis ne postoji.
</InfoBox>

## 6. Dohvat podataka u `OnInitializedAsync`

Sada stranicu spajamo sa servisom.

```razor
@code {
    [Parameter]
    public int Id { get; set; }

    private Animal? animal;
    private bool isLoading = true;

    protected override async Task OnInitializedAsync()
    {
        animal = await AnimalService.GetAnimalByIdAsync(Id);
        isLoading = false;
    }
}
```

Ovdje je logika jednostavna i jasna:

- stranica primi ID iz URL-a
- pozove servis
- servis vrati jednu životinju s relacijama
- prestajemo prikazivati loading stanje

<InfoBox type="warning" title="Česta greška">
Učenici ponekad zaborave da metoda mora biti <code>async</code> i da mora koristiti <code>await</code>. Druga česta greška je da zovu pogrešnu metodu servisa, npr. onu za dohvat svih životinja umjesto jedne konkretne.
</InfoBox>

## 10. Kartica s osnovnim podacima

Sada u prvu karticu stavljamo podatke koji su najvažniji za svakodnevni pregled.

```razor
<MudPaper Class="pa-4 mb-4" Elevation="2">
    <MudText Typo="Typo.h6" Class="mb-3">Osnovni podaci</MudText>

    <MudGrid>
        <MudItem xs="12" md="6">
            <MudText><strong>Vrsta:</strong> @animal.Species?.Name</MudText>
        </MudItem>
        <MudItem xs="12" md="6">
            <MudText><strong>Pasmina:</strong> @(animal.Breed?.Name ?? "Nije odabrana")</MudText>
        </MudItem>
        <MudItem xs="12" md="6">
            <MudText><strong>Spol:</strong> @animal.Gender</MudText>
        </MudItem>
        <MudItem xs="12" md="6">
            <MudText><strong>Dob:</strong> @animal.Age</MudText>
        </MudItem>
        <MudItem xs="12" md="6">
            <MudText><strong>Datum dolaska:</strong> @animal.ArrivalDate.ToShortDateString()</MudText>
        </MudItem>
        <MudItem xs="12" md="6">
            <MudText><strong>Broj čipa:</strong> @(string.IsNullOrWhiteSpace(animal.ChipNumber) ? "Nije unesen" : animal.ChipNumber)</MudText>
        </MudItem>
    </MudGrid>
</MudPaper>
```

<InfoBox type="info" title='Zašto koristimo `??` i provjere praznog teksta?'>
U stvarnim aplikacijama svi podaci nisu uvijek popunjeni. Zato details stranica mora znati prikazati i situacije u kojima nešto nije odabrano ili uneseno. To je profesionalniji pristup od prikaza praznih vrijednosti ili ružnih grešaka.
</InfoBox>

## 17. Najčešće greške

<InfoBox type="warning" title="Greška 1 — Zaboravljen Include">
Ako učenik ne doda `Include`, details stranica može prikazivati praznu vrstu, pasminu ili status iako su strani ključevi ispravno postavljeni.
</InfoBox>

<InfoBox type="warning" title="Greška 2 — Pogrešna ruta ili ID">
Ako ruta nije ispravna ili gumb šalje pogrešan ID, stranica neće pronaći zapis.
</InfoBox>

<InfoBox type="warning" title="Greška 3 — Sve stavljeno u jednu veliku karticu">
Ako se svi podaci naguraju bez rasporeda, details stranica gubi smisao. Cilj je preglednost, ne samo funkcionalnost.
</InfoBox>

<InfoBox type="warning" title="Greška 4 — Miješanje details i edit logike">
Details stranica služi pregledu. Ako učenik odmah u nju krene dodavati hrpu input polja, vraćamo se na neuredan pristup koji smo htjeli izbjeći.
</InfoBox>

## 18. Vježbe za učenike

- pasminu

## 19. Domaći rad

Za domaći učenici trebaju:

1. urediti details stranicu tako da izgleda pregledno i profesionalno
2. prilagoditi boje statusa, npr.:
   - dostupna → zelena
   - rezervirana → narančasta
   - udomljena → plava ili siva
3. dodati još barem jedno polje po vlastitom izboru i prikazati ga na details stranici
4. testirati radi li stranica ispravno za više različitih zapisa iz baze
