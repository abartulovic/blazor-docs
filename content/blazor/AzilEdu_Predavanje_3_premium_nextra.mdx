## 2. Što je model i zašto nam sada treba

Prije nego išta prikažemo na ekranu, moramo znati <strong>što točno prikazujemo</strong>.

Ako želimo prikazati životinje u azilu, moramo definirati koje informacije o svakoj životinji postoje u aplikaciji. To radimo pomoću modela.

### Što je model?

Model je klasa koja opisuje neki pojam iz stvarnog sustava.

U našem slučaju pojam je:

- životinja u azilu

To znači da model može sadržavati podatke kao što su:

- identifikator
- ime životinje
- vrsta
- pasmina
- starost
- datum dolaska
- status udomljenja

<InfoBox type="warning" title="Česta pogreška učenika">
  Učenici često misle da je stranica isto što i model. To nije točno.
  <br /><br />
  <strong>Model</strong> opisuje podatke.<br />
  <strong>Stranica</strong> prikazuje te podatke korisniku.<br />
  <strong>Komponenta</strong> pomaže da se ti podaci prikažu uredno i pregledno.
</InfoBox>

## 3. Gdje ćemo smjestiti model `Animal`

U ovom projektu koristimo više projekata unutar istog rješenja. Zato trebamo razmisliti gdje model treba biti smješten.

Budući da će podatke o životinji trebati i korisnički dio aplikacije i eventualno API dio, model ima smisla držati u projektu koji je zajednički.

Najčešće će to biti:

```text
Shared/Models/Animal.cs
```

Ako u tvojoj postojećoj strukturi već postoji drugi folder za zajedničke modele, prilagodi putanju toj organizaciji. Važno je da učenici razumiju <strong>zašto</strong> model ide u zajednički dio projekta.

### Zašto u `Shared`?

Zato što isti model može trebati:

- Blazor aplikaciji za prikaz
- API dijelu za slanje podataka
- kasnije i drugim dijelovima projekta

To smanjuje dupliranje i čini projekt urednijim.

## 4. Izrada modela `Animal`

Sada izrađujemo osnovnu klasu koja opisuje jednu životinju u azilu.

```csharp
namespace AzilEdu.Shared.Models;

public class Animal
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string Species { get; set; } = string.Empty;
    public string Breed { get; set; } = string.Empty;
    public int Age { get; set; }
    public DateTime ArrivalDate { get; set; }
    public bool IsAdopted { get; set; }
}
```

### Objašnjenje svojstava

- `Id` – jedinstveni identifikator životinje
- `Name` – ime životinje
- `Species` – vrsta, npr. pas ili mačka
- `Breed` – pasmina
- `Age` – starost u godinama
- `ArrivalDate` – datum dolaska u azil
- `IsAdopted` – označava je li životinja udomljena

<InfoBox type="info" title="Zašto odmah koristimo više svojstava?">
  Mogli bismo početi i s manjim brojem polja, ali ovdje je korisno da učenici odmah vide kako jedan model iz stvarnog sustava obično sadrži više povezanih informacija. Tako bolje razumiju smisao aplikacije i lakše kasnije prelaze na unos, uređivanje i spremanje podataka.
</InfoBox>

## 5. Priprema probnih podataka

U ovoj fazi još ne radimo s bazom podataka niti s API dohvatom. To dolazi kasnije. Sada želimo da učenici prvo razumiju kako se podaci <strong>prikazuju</strong>.

Zato ćemo privremeno napraviti listu probnih podataka direktno na stranici.

To je dobar pristup jer:

- učenici vide rezultat odmah
- nema dodatnog opterećenja bazom i API pozivima
- lakše se fokusiraju na model i prikaz

### Primjer probnih podataka

```csharp

    {

        Species = "Pas",
```

<InfoBox type="warning" title="Zašto ovo nije trajno rješenje?">

## 6. Izrada nove stranice za prikaz životinja

Sada trebamo napraviti novu Blazor stranicu koja će prikazivati popis životinja.

Primjer putanje može biti:

```text
Components/Pages/Animals.razor
```

Ako u projektu koristiš nešto drukčiju organizaciju stranica, zadrži logiku koju već imate. Najvažnije je da stranica bude dio iste aplikacije i da je učenici mogu lako pronaći.

### Osnovni početak stranice

```razor
@page "/animals"
@using AzilEdu.Shared.Models

<PageTitle>Životinje</PageTitle>

<MudText Typo="Typo.h4" Class="mb-4">Popis životinja</MudText>
```

### Što ovdje radimo?

- `@page "/animals"` određuje URL stranice
- `@using AzilEdu.Shared.Models` omogućuje korištenje modela `Animal`
- `PageTitle` mijenja naslov kartice preglednika
- `MudText` prikazuje naslov stranice koristeći MudBlazor tipografiju

## 7. Prikaz podataka pomoću `MudTable`

Sada dolazimo do dijela gdje MudBlazor stvarno pokazuje svoju vrijednost. Želimo uredno prikazati više zapisa, a tablica je za to vrlo dobar početni izbor.

Kod tablice će imati <strong>preview s desne strane</strong> tek kada dodaš screenshot. Za sada ostavljamo točno opisano mjesto slike.

<CodePreviewSplit
  title="Osnovni prikaz popisa životinja u MudTable komponenti"
  language="razor"
  image="/images/aziledu/predavanje-3/previews/mudtable-animals-basic.webp"
  imageAlt="Pregled osnovne MudTable tablice s popisom životinja"
  imageCaption="Dodaj screenshot u: /images/aziledu/predavanje-3/previews/mudtable-animals-basic.webp"
  code={`<MudTable Items="@animals" Hover="true" Striped="true" Bordered="true">
    <HeaderContent>
        <MudTh>ID</MudTh>
        <MudTh>Ime</MudTh>
        <MudTh>Vrsta</MudTh>
        <MudTh>Pasmina</MudTh>
        <MudTh>Starost</MudTh>
        <MudTh>Datum dolaska</MudTh>
        <MudTh>Status</MudTh>
    </HeaderContent>
    <RowTemplate>
        <MudTd DataLabel="ID">@context.Id</MudTd>
        <MudTd DataLabel="Ime">@context.Name</MudTd>
        <MudTd DataLabel="Vrsta">@context.Species</MudTd>
        <MudTd DataLabel="Pasmina">@context.Breed</MudTd>
        <MudTd DataLabel="Starost">@context.Age</MudTd>
        <MudTd DataLabel="Datum dolaska">@context.ArrivalDate.ToShortDateString()</MudTd>
        <MudTd DataLabel="Status">
            @(context.IsAdopted ? "Udomljena" : "U azilu")
        </MudTd>
    </RowTemplate>
</MudTable>`}
/>

### Objašnjenje tablice

- `Items="@animals"` govori tablici da koristi listu `animals`
- `HeaderContent` definira zaglavlje tablice
- `RowTemplate` opisuje kako izgleda svaki redak
- `@context` predstavlja jedan objekt iz liste

### Zašto je `MudTable` dobar izbor?

Zato što učenici odmah vide:

- kako više objekata iz liste postaje više redaka u tablici
- kako se svako svojstvo modela može prikazati u posebnom stupcu
- kako MudBlazor omogućuje pregledan i uredan prikaz bez puno dodatnog CSS-a

<InfoBox type="info" title="Što znači @context?">
  U `RowTemplate` dijelu `@context` predstavlja trenutačni objekt koji se obrađuje. Budući da tablica prolazi kroz listu `animals`, svaki red koristi podatke jedne životinje. Zato `@context.Name` prikazuje ime trenutačne životinje, `@context.Species` vrstu i tako dalje.
</InfoBox>

## 9. Malo bolje isticanje statusa uz `MudChip`

Kada učenici vide osnovnu tablicu, dobro je pokazati kako se mali detalji u sučelju mogu vizualno poboljšati. Jedan dobar primjer je status udomljenja.

Umjesto običnog teksta, možemo koristiti `MudChip`.

<CodePreviewSplit
  title="Vizualno isticanje statusa pomoću MudChip komponente"
  language="razor"
  image="/images/aziledu/predavanje-3/previews/mudchip-adoption-status.webp"
  imageAlt="Prikaz statusa životinje kroz MudChip komponente"
  imageCaption="Dodaj screenshot u: /images/aziledu/predavanje-3/previews/mudchip-adoption-status.webp"
  code={`<MudTd DataLabel="Status">
    @if (context.IsAdopted)
    {
        <MudChip Color="Color.Success" Variant="Variant.Filled">
            Udomljena
        </MudChip>
    }
    else
    {
        <MudChip Color="Color.Warning" Variant="Variant.Filled">
            U azilu
        </MudChip>
    }
</MudTd>`}
/>

### Zašto je ovo korisno?

Zato što učenici vide da komponente nisu samo "ukras", nego pomažu korisniku da brže razumije stanje podataka. Zelena oznaka odmah sugerira pozitivan status, dok druga boja jasno razlikuje životinje koje još čekaju udomljenje.

<InfoBox type="success" title="Dobra praksa u UI-u">
  Kada je neki podatak važan, korisno ga je vizualno istaknuti. U ovom slučaju status udomljenja je važna informacija i zato ga ima smisla prikazati upečatljivije od običnog teksta.
</InfoBox>

## 12. Moguće greške i problemi

### Problem 1: Stranica se ne prikazuje

Provjeri:

- postoji li `@page "/animals"`
- je li datoteka spremljena
- vodi li navigacija na točnu putanju

### Problem 2: `Animal` nije prepoznat

Provjeri:

- je li dodan `@using AzilEdu.Shared.Models`
- je li model stvarno u projektu `Shared`
- je li namespace točan

### Problem 3: Tablica je prazna

Provjeri:

- postoji li lista `animals`
- sadrži li podatke
- je li `Items="@animals"` ispravno napisano

### Problem 4: Datum ili status izgledaju čudno

Provjeri:

- koristi li se ispravno `ToShortDateString()`
- je li logika za `IsAdopted` dobro napisana

### Problem 5: Navigacijski link postoji, ali ne radi

Provjeri:

- odgovara li `Href` stvarnoj `@page` putanji
- je li link dodan u pravu navigacijsku komponentu
