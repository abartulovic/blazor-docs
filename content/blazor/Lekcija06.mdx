## 1. Gdje smo stali

Na kraju prethodne lekcije <code>Animals</code> stranica je već radila ispravno:

- App je slao GET zahtjev preko <code>HttpClient</code>
- API kontroler je čitao bazu preko <code>DbContext</code>
- kontroler je podatke mapirao u DTO
- App je prikazivao listu u MudTable komponenti

Sada idemo korak dalje. Nije dovoljno samo vidjeti postojeće podatke. Stvarna aplikacija mora moći:

- unijeti novu životinju
- urediti postojeći zapis
- obrisati zapis koji više nije potreban

## 4. Proširenje `AnimalsController` s POST, PUT i DELETE metodama

Kontroler iz prethodne lekcije već ima GET metodu. Sada ga proširujemo tako da može primati nove podatke i mijenjati postojeće.

### Korak 1 – POST metoda za unos nove životinje

```csharp
[HttpPost]
public async Task<ActionResult<AnimalDto>> CreateAnimal(SaveAnimalDto dto)
{
    var animal = new Animal
    {
        Name = dto.Name,
        Species = dto.Species,
        Breed = dto.Breed,
        Gender = dto.Gender,
        Age = dto.Age,
        ArrivalDate = dto.ArrivalDate,
        IsAdopted = dto.IsAdopted,
        ImageUrl = dto.ImageUrl,
        Description = dto.Description
    };

    _context.Animals.Add(animal);
    await _context.SaveChangesAsync();

    var result = new AnimalDto
    {
        Id = animal.Id,
        Name = animal.Name,
        Species = animal.Species,
        Breed = animal.Breed,
        Gender = animal.Gender,
        Age = animal.Age,
        ArrivalDate = animal.ArrivalDate,
        IsAdopted = animal.IsAdopted,
        ImageUrl = animal.ImageUrl,
        Description = animal.Description
    };

    return CreatedAtAction(nameof(GetAnimalById), new { id = animal.Id }, result);
}
```

#### Što znači `[HttpPost]`?

Ovaj atribut govori da se metoda pokreće kada API primi **POST zahtjev** na rutu kontrolera, u ovom slučaju:

```text
api/animals
```

POST koristimo kada želimo **poslati nove podatke** prema serveru i iz njih stvoriti novi zapis u bazi.

#### Što znači potpis metode?

```csharp
public async Task<ActionResult<AnimalDto>> CreateAnimal(SaveAnimalDto dto)
```

- `CreateAnimal` je naziv metode
- `SaveAnimalDto dto` predstavlja podatke koje App šalje iz forme prema API-ju
- `ActionResult<AnimalDto>` znači da metoda može vratiti DTO objekt, ali i neki HTTP status ako bude potrebno
- `async` i `Task` znače da metoda radi asinkrono, što je važno jer pristup bazi i spremanje ne želimo izvršavati blokirajuće

Drugim riječima: ova metoda prima podatke iz forme, obrađuje ih i na kraju vraća odgovor koji App može razumjeti.

#### Zašto prvo radimo `new Animal`?

```csharp
var animal = new Animal
{
    Name = dto.Name,
    Species = dto.Species,
    Breed = dto.Breed,
    Gender = dto.Gender,
    Age = dto.Age,
    ArrivalDate = dto.ArrivalDate,
    IsAdopted = dto.IsAdopted,
    ImageUrl = dto.ImageUrl,
    Description = dto.Description
};
```

Ovdje iz DTO objekta stvaramo pravi entity model `Animal` koji EF Core poznaje i koji se sprema u tablicu `Animals`. DTO služi za prijenos podataka između App-a i API-ja, a entity predstavlja zapis koji EF Core prati u radu s bazom.

#### Što radi `_context.Animals.Add(animal);`?

```csharp
_context.Animals.Add(animal);
```

Ova linija **ne sprema još ništa u bazu**. Ona samo govori EF Core-u: “Ovaj novi objekt treba dodati u tablicu `Animals`.” EF Core tada označava objekt `animal` kao novi zapis koji treba ubaciti u bazu pri sljedećem spremanju.

#### Što radi `await _context.SaveChangesAsync();`?

```csharp
await _context.SaveChangesAsync();
```

Tek ova linija stvarno izvršava promjenu u bazi. EF Core šalje SQL naredbu prema SQLite bazi i novi red se upisuje u tablicu. Nakon toga objekt `animal` više nije samo objekt u memoriji, nego stvarni zapis u bazi, a često tada dobije i novi `Id`.

#### Zašto nakon spremanja radimo `var result = new AnimalDto`?

```csharp
var result = new AnimalDto
{
    Id = animal.Id,
    Name = animal.Name,
    Species = animal.Species,
    Breed = animal.Breed,
    Gender = animal.Gender,
    Age = animal.Age,
    ArrivalDate = animal.ArrivalDate,
    IsAdopted = animal.IsAdopted,
    ImageUrl = animal.ImageUrl,
    Description = animal.Description
};
```

Nakon spremanja ne vraćamo direktno entity `Animal`, nego stvaramo novi `AnimalDto`. Time zadržavamo isti obrazac kao i u prethodnoj lekciji: API prema App-u vraća DTO objekte, a ne interne EF Core entitete. To je čišće, sigurnije i dosljedno.

#### Što znači `return CreatedAtAction(...)`?

```csharp
return CreatedAtAction(nameof(GetAnimalById), new { id = animal.Id }, result);
```

Ovo vraća HTTP status **201 Created**, što je standardni odgovor kada je novi zapis uspješno stvoren.

Ovaj povrat dodatno govori dvije stvari:

- `nameof(GetAnimalById)` upućuje na metodu kojom se može dohvatiti upravo stvoreni zapis
- `new { id = animal.Id }` kaže koji ID treba koristiti u toj ruti
- `result` je DTO objekt koji se šalje natrag u odgovoru

To znači da API ne kaže samo “uspjelo je”, nego i vraća informaciju gdje se novostvoreni zapis nalazi i koji su njegovi podaci nakon spremanja.

### Korak 2 – GET metoda za dohvat jedne životinje po ID-u

Ova metoda služi za dohvat jednog konkretnog zapisa po identifikatoru. Posebno je važna kod stranice za uređivanje, jer prije prikaza forme moramo dohvatiti postojeće podatke iz baze.

```csharp
[HttpGet("{id}")]
public async Task<ActionResult<AnimalDto>> GetAnimalById(int id)
{
    var animal = await _context.Animals.FindAsync(id);

    if (animal is null)
        return NotFound();

    var dto = new AnimalDto
    {
        Id = animal.Id,
        Name = animal.Name,
        Species = animal.Species,
        Breed = animal.Breed,
        Gender = animal.Gender,
        Age = animal.Age,
        ArrivalDate = animal.ArrivalDate,
        IsAdopted = animal.IsAdopted,
        ImageUrl = animal.ImageUrl,
        Description = animal.Description
    };

    return Ok(dto);
}
```

#### Što znači `[HttpGet("{id}")]`?

Ovaj atribut govori da se metoda pokreće kada API primi **GET zahtjev** na rutu poput:

```text
api/animals/5
```

Dio `{id}` znači da iz URL-a uzimamo vrijednost identifikatora i spremamo je u parametar `int id`.

#### Što znači potpis metode?

```csharp
public async Task<ActionResult<AnimalDto>> GetAnimalById(int id)
```

- `GetAnimalById` je naziv metode
- `int id` je identifikator zapisa koji tražimo
- `ActionResult<AnimalDto>` znači da metoda može vratiti ili DTO objekt ili neki HTTP status, npr. `NotFound()`
- `async` i `Task` znače da metoda asinkrono čeka rad s bazom

#### Što radi `FindAsync(id)`?

```csharp
var animal = await _context.Animals.FindAsync(id);
```

Ovdje EF Core pokušava pronaći zapis u tablici `Animals` prema primarnom ključu. Ako zapis postoji, vraća objekt `Animal`. Ako ne postoji, rezultat je `null`.

#### Zašto provjeravamo `if (animal is null)`?

Ako zapis s tim ID-em ne postoji, nema smisla nastaviti s mapiranjem u DTO. Zato vraćamo:

```csharp
return NotFound();
```

To znači HTTP status **404 Not Found**.

#### Zašto opet radimo `new AnimalDto`?

API ne vraća direktno entity model baze, nego DTO. Time zadržavamo isti obrazac kao u prethodnoj lekciji: baza i EF Core ostaju u API sloju, a App prima samo podatke koji su pripremljeni za prijenos.

#### Što znači `return Ok(dto);`?

To vraća HTTP status **200 OK** i DTO objekt u tijelu odgovora. Upravo taj DTO kasnije u App projektu možeš dohvatiti pomoću `GetFromJsonAsync<AnimalDto>(...)`.

### Korak 3 – PUT metoda za uređivanje postojećeg zapisa

PUT metoda služi za ažuriranje već postojećeg zapisa. App šalje nove vrijednosti, a API pronalazi odgovarajuću životinju i upisuje promjene u bazu.

```csharp
[HttpPut("{id}")]
public async Task<IActionResult> UpdateAnimal(int id, SaveAnimalDto dto)
{
    var animal = await _context.Animals.FindAsync(id);

    if (animal is null)
        return NotFound();

    animal.Name = dto.Name;
    animal.Species = dto.Species;
    animal.Breed = dto.Breed;
    animal.Gender = dto.Gender;
    animal.Age = dto.Age;
    animal.ArrivalDate = dto.ArrivalDate;
    animal.IsAdopted = dto.IsAdopted;
    animal.ImageUrl = dto.ImageUrl;
    animal.Description = dto.Description;

    await _context.SaveChangesAsync();
    return NoContent();
}
```

#### Što znači `[HttpPut("{id}")]`?

To znači da se metoda pokreće za **PUT zahtjev** na rutu poput:

```text
api/animals/5
```

`PUT` koristimo kada želimo zamijeniti ili ažurirati postojeći zapis.

#### Što znači potpis metode?

```csharp
public async Task<IActionResult> UpdateAnimal(int id, SaveAnimalDto dto)
```

- `int id` dolazi iz URL-a i govori koji zapis uređujemo
- `SaveAnimalDto dto` dolazi iz tijela HTTP zahtjeva i nosi nove vrijednosti
- `IActionResult` znači da metoda vraća HTTP odgovor, npr. `NotFound()` ili `NoContent()`

#### Zašto opet prvo tražimo postojeći zapis?

```csharp
var animal = await _context.Animals.FindAsync(id);
```

Ne možemo ažurirati zapis koji ne postoji. Zato prvo provjeravamo postoji li životinja s tim ID-em.

#### Što se događa u ovim linijama?

```csharp
animal.Name = dto.Name;
animal.Species = dto.Species;
...
animal.Description = dto.Description;
```

Ovdje prepisujemo postojeći entity novim vrijednostima iz DTO objekta. EF Core već prati objekt `animal`, pa nakon promjene njegovih svojstava zna što treba ažurirati u bazi.

#### Što radi `await _context.SaveChangesAsync();` u PUT metodi?

I ovdje ta linija znači stvarno spremanje promjena u bazu. Bez nje bi promjene postojale samo u memoriji aplikacije i ne bi završile u tablici `Animals`.

#### Zašto vraćamo `NoContent()`?

```csharp
return NoContent();
```

To znači HTTP status **204 No Content**. Time API govori: promjena je uspješno spremljena, ali nije potrebno vraćati dodatni sadržaj u odgovoru.

### Korak 4 – DELETE metoda za brisanje

DELETE metoda služi za uklanjanje postojećeg zapisa iz baze.

```csharp
[HttpDelete("{id}")]
public async Task<IActionResult> DeleteAnimal(int id)
{
    var animal = await _context.Animals.FindAsync(id);

    if (animal is null)
        return NotFound();

    _context.Animals.Remove(animal);
    await _context.SaveChangesAsync();

    return NoContent();
}
```

#### Što znači `[HttpDelete("{id}")]`?

To znači da se metoda pokreće kada API primi **DELETE zahtjev** na rutu poput:

```text
api/animals/5
```

Time jasno govorimo da želimo obrisati zapis s određenim ID-em.

#### Zašto opet radimo `FindAsync(id)`?

I kod brisanja prvo moramo pronaći zapis. Ako zapis ne postoji, nema što brisati i vraćamo `NotFound()`.

#### Što radi `Remove(animal)`?

```csharp
_context.Animals.Remove(animal);
```

Time EF Core označava pronađeni objekt kao zapis koji treba obrisati iz baze.

#### Zašto nakon toga opet ide `SaveChangesAsync()`?

Kao i kod unosa i uređivanja, promjena se stvarno izvršava tek kada pozovemo:

```csharp
await _context.SaveChangesAsync();
```

Tek tada se u bazi izvršava brisanje.

#### Zašto metoda vraća `NoContent()`?

Ako je brisanje uspjelo, API vraća **204 No Content**. To je čest i ispravan odgovor za DELETE operacije kada nema potrebe vraćati dodatni objekt u odgovoru.

### Cjeloviti sadržaj kontrolera nakon nadogradnje

```csharp
using AzilEdu.Api.Data;
using AzilEdu.Shared.DTOs;
using AzilEdu.Shared.Models;
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;

namespace AzilEdu.Api.Controllers;

[ApiController]
[Route("api/[controller]")]
public class AnimalsController : ControllerBase
{
    private readonly AzilEduDbContext _context;

    public AnimalsController(AzilEduDbContext context)
    {
        _context = context;
    }

    [HttpGet]
    public async Task<ActionResult<List<AnimalDto>>> GetAnimals()
    {
        var animals = await _context.Animals
            .OrderBy(a => a.Name)
            .Select(a => new AnimalDto
            {
                Id = a.Id,
                Name = a.Name,
                Species = a.Species,
                Breed = a.Breed,
                Gender = a.Gender,
                Age = a.Age,
                ArrivalDate = a.ArrivalDate,
                IsAdopted = a.IsAdopted,
                ImageUrl = a.ImageUrl,
                Description = a.Description
            })
            .ToListAsync();

        return Ok(animals);
    }

    [HttpGet("{id}")]
    public async Task<ActionResult<AnimalDto>> GetAnimalById(int id)
    {
        var animal = await _context.Animals.FindAsync(id);

        if (animal is null)
            return NotFound();

        var dto = new AnimalDto
        {
            Id = animal.Id,
            Name = animal.Name,
            Species = animal.Species,
            Breed = animal.Breed,
            Gender = animal.Gender,
            Age = animal.Age,
            ArrivalDate = animal.ArrivalDate,
            IsAdopted = animal.IsAdopted,
            ImageUrl = animal.ImageUrl,
            Description = animal.Description
        };

        return Ok(dto);
    }

    [HttpPost]
    public async Task<ActionResult<AnimalDto>> CreateAnimal(SaveAnimalDto dto)
    {
        var animal = new Animal
        {
            Name = dto.Name,
            Species = dto.Species,
            Breed = dto.Breed,
            Gender = dto.Gender,
            Age = dto.Age,
            ArrivalDate = dto.ArrivalDate,
            IsAdopted = dto.IsAdopted,
            ImageUrl = dto.ImageUrl,
            Description = dto.Description
        };

        _context.Animals.Add(animal);
        await _context.SaveChangesAsync();

        var result = new AnimalDto
        {
            Id = animal.Id,
            Name = animal.Name,
            Species = animal.Species,
            Breed = animal.Breed,
            Gender = animal.Gender,
            Age = animal.Age,
            ArrivalDate = animal.ArrivalDate,
            IsAdopted = animal.IsAdopted,
            ImageUrl = animal.ImageUrl,
            Description = animal.Description
        };

        return CreatedAtAction(nameof(GetAnimalById), new { id = animal.Id }, result);
    }

    [HttpPut("{id}")]
    public async Task<IActionResult> UpdateAnimal(int id, SaveAnimalDto dto)
    {
        var animal = await _context.Animals.FindAsync(id);

        if (animal is null)
            return NotFound();

        animal.Name = dto.Name;
        animal.Species = dto.Species;
        animal.Breed = dto.Breed;
        animal.Gender = dto.Gender;
        animal.Age = dto.Age;
        animal.ArrivalDate = dto.ArrivalDate;
        animal.IsAdopted = dto.IsAdopted;
        animal.ImageUrl = dto.ImageUrl;
        animal.Description = dto.Description;

        await _context.SaveChangesAsync();
        return NoContent();
    }

    [HttpDelete("{id}")]
    public async Task<IActionResult> DeleteAnimal(int id)
    {
        var animal = await _context.Animals.FindAsync(id);

        if (animal is null)
            return NotFound();

        _context.Animals.Remove(animal);
        await _context.SaveChangesAsync();

        return NoContent();
    }
}
```

### Korak 3 – slanje POST zahtjeva

> `MudDatePicker` u ovoj formi vežemo na pomoćno nullable polje `arrivalDate`, a ne direktno na `model.ArrivalDate`, jer komponenta očekuje `DateTime?`. Tako izbjegavamo greške pri parsiranju forme i zadržavamo kontrolu nad vrijednošću koju šaljemo prema API-ju.


```razor
@code {
    private SaveAnimalDto model = new()
    {
        ArrivalDate = DateTime.Today,
        ImageUrl = "/images/animals/placeholder.webp"
    };

    private DateTime? arrivalDate = DateTime.Today;

    private async Task SaveAnimal()
    {
        model.ArrivalDate = arrivalDate;

        var response = await Http.PostAsJsonAsync("api/animals", model);

        if (response.IsSuccessStatusCode)
        {
            Navigation.NavigateTo("/animals");
        }
    }
}
```

### Kako radi DELETE zahtjev?

Kada klikneš gumb `Obriši`, Blazor App poziva:

```razor
await Http.DeleteAsync($"api/animals/{id}");
```

Taj poziv šalje HTTP DELETE zahtjev na rutu poput `api/animals/5`.

Na API strani metoda:

```csharp
[HttpDelete("{id}")]
public async Task<IActionResult> DeleteAnimal(int id)
```

radi sljedeće:

- iz URL-a prima `id`
- traži odgovarajući zapis u bazi
- ako zapis ne postoji, vraća `NotFound()`
- ako zapis postoji, uklanja ga iz konteksta pomoću `_context.Animals.Remove(animal);`
- `await _context.SaveChangesAsync();` trajno briše zapis iz baze
- `return NoContent();` javlja da je brisanje uspjelo

Nakon uspješnog DELETE zahtjeva, App odmah ažurira lokalnu listu i zato se obrisani redak više ne prikazuje u tablici.

### Napomena o potvrdi prije brisanja

U ozbiljnijoj verziji bilo bi dobro prije brisanja tražiti potvrdu kroz dijalog. Ovdje zadržavamo jednostavniju varijantu da prvo razumiješ osnovni tok DELETE zahtjeva.

## 10. Domaći rad: CUD za `HousingUnits`

Domaći rad se direktno nastavlja na prethodni domaći u kojem je modul `HousingUnits` već imao DTO i GET dohvat preko API-ja. Sada na isti modul treba prenijeti puni CUD obrazac koji smo na satu napravili za `Animals`.

### Datoteke koje trebaš napraviti ili urediti

1. `Shared/DTOs/SaveHousingUnitDto.cs`
2. `AzilEdu.Api/Controllers/HousingUnitsController.cs`
3. `AzilEdu.App/Components/Pages/HousingUnits.razor`
4. `AzilEdu.App/Components/Pages/HousingUnitsCreate.razor`
5. `AzilEdu.App/Components/Pages/HousingUnitsEdit.razor`

### Predloženi DTO za spremanje

```csharp
namespace AzilEdu.Shared.DTOs;

public class SaveHousingUnitDto

    public string ImageUrl { get; set; } = string.Empty;
```

### Točno što treba napraviti

#### 1. U API kontroleru dodaj pune CUD metode

U `HousingUnitsController` napravi ili dovrši ove metode:

- `GET` za listu
- `GET by id`
- `POST`
- `PUT`
- `DELETE`

Svaka metoda treba koristiti isti obrazac kao u `AnimalsController`:
- API prima `SaveHousingUnitDto`
- po potrebi radi mapiranje u entity
- sprema promjene preko `_context.SaveChangesAsync()`
- vraća odgovarajući HTTP odgovor

#### 2. Napravi stranicu `HousingUnitsCreate.razor`

Na toj stranici treba postojati forma za unos nove smještajne jedinice. Forma mora imati polja:

- naziv jedinice
- tip jedinice
- kapacitet
- broj zauzetih mjesta
- status `IsFull`
- putanju slike

Nakon klika na gumb za spremanje stranica mora poslati **POST zahtjev** prema API-ju i nakon uspješnog unosa vratiti se na listu `HousingUnits`.

#### 3. Napravi stranicu `HousingUnitsEdit.razor`

Na toj stranici treba:
- iz URL-a uzeti ID
- dohvatiti postojeći zapis preko `GET by id`
- popuniti formu postojećim vrijednostima
- poslati promjene prema API-ju preko **PUT zahtjeva**
- nakon uspješnog spremanja vratiti se na listu

#### 4. Nadogradi `HousingUnits.razor`

Na postojećoj listi treba dodati:
- gumb `Nova jedinica`
- stupac `Akcije`
- gumb `Uredi`
- gumb `Obriši`

Klik na `Obriši` mora slati **DELETE zahtjev** prema API-ju i nakon uspjeha odmah maknuti zapis s liste ili ponovno učitati podatke.

### Kako gotov rad mora izgledati

Na kraju domaćeg mora biti moguće:

1. otvoriti listu `HousingUnits`
2. kliknuti `Nova jedinica`
3. unijeti novu jedinicu i spremiti je u bazu
4. otvoriti postojeću jedinicu kroz `Uredi`
5. promijeniti podatke i spremiti ih
6. obrisati jedinicu preko gumba `Obriši`
7. vidjeti da su se promjene stvarno zapisale u bazi i na listi

### Minimalni uvjet za potpuno rješenje

Domaći se smatra dovršenim tek kada:
- sve tri operacije `POST`, `PUT` i `DELETE` rade bez greške
- postoji DTO za spremanje
- App ne pristupa direktno bazi nego sve ide preko API-ja
- forme i lista vizualno prate stil ostatka aplikacije

## 11. Sažetak lekcije

U ovoj lekciji složili smo puni CUD tok za modul `Animals`.

- **Create** radi tako da App preko forme šalje `POST` zahtjev s `SaveAnimalDto` objektom, API iz tog DTO-a stvara `Animal` entity, sprema ga u bazu i vraća odgovor `201 Created`.
- **Read by id** služi za dohvat jednog zapisa. Taj korak je posebno važan kod uređivanja jer prvo moramo napuniti formu postojećim podacima.
- **Update** radi tako da App pošalje `PUT` zahtjev s novim vrijednostima, API pronađe postojeći zapis, promijeni njegova svojstva i spremi promjene u bazu.
- **Delete** radi tako da App pošalje `DELETE` zahtjev, API pronađe zapis, označi ga za brisanje i nakon `SaveChangesAsync()` ukloni ga iz baze.
- Na Razor stranicama `AnimalsCreate.razor` i `AnimalsEdit.razor` forma ne razgovara s bazom direktno, nego uvijek preko `HttpClient` i API kontrolera.
- Isti obrazac sada treba samostalno prenijeti na modul `HousingUnits`.
