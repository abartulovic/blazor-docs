## Zašto uopće dijelimo projekt na više dijelova?

Početnici često naprave jednu veliku aplikaciju bez jasne granice između korisničkog sučelja, poslovne logike i modela podataka. To kratkoročno izgleda brže, ali dugoročno stvara nered.

U AzilEdu projektu odmah uvodimo red:

  <MiniCard title="AzilEdu.App">

  </MiniCard>

Tok komunikacije kasnije će izgledati ovako:

```text
Korisnik -> AzilEdu.App -> AzilEdu.Api -> baza/podaci -> odgovor -> AzilEdu.App
```

<Box title="Zapamti" tone="blue">

  <strong>Api nije isto što i baza.</strong><br />
  <strong>Shared nije mjesto za Razor stranice ni kontrolere.</strong>
</Box>

## Pojmovi koje moraš razumjeti prije rada

### Solution

<strong>Solution</strong> je krovno rješenje. U njemu možeš imati jedan ili više projekata.

### Project

<strong>Project</strong> je jedan konkretan dio rješenja. U ovoj lekciji imat ćemo tri projekta.

### Frontend

<strong>Frontend</strong> je ono što korisnik vidi u pregledniku.

### Backend / API

<strong>Backend</strong> prima zahtjeve, obrađuje ih i vraća podatke. Kod nas je to <strong>ASP.NET Core Web API</strong>.

### Shared

<strong>Shared</strong> je zajednička biblioteka u kojoj će se nalaziti DTO modeli i drugi zajednički tipovi koje koriste i frontend i backend.

Kad završiš lekciju, moraš znati prepoznati ove dijelove:
